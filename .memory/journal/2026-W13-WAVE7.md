## Key Fixes Applied

### Fix 1: Tab Filter Animation Bug
- **Issue:** First load showed nothing; switching tabs loaded only one tab; returning to previous tab showed nothing
- **Root Cause:** Framer Motion cascade animation had initial state at "visible", so component updates didn't re-trigger
- **Fix:** Added `key={activeTab}` to TreatmentsSimple and TreatmentsComplex to force remount
- **Result:** Animations now correctly reset on tab change

### Fix 2: Edit Modal Not Opening
- **Issue:** Clicking treatment didn't open edit form
- **Root Cause:** ProtocolRow had no onEdit callback; TreatmentsRedesign wasn't wired for editing
- **Fix:**
  1. Created `handleEditProtocol()` in TreatmentsRedesign that fetches full protocol via `protocolService.getById()`
  2. Passed to both TreatmentsSimple and TreatmentsComplex
  3. Updated AnvisaSearchBar to call `onEditProtocol` for existing protocols
- **Result:** Clicking any treatment opens edit form

### Fix 3: Groups Not Computing Correctly
- **Issue:** Treatments not grouped by treatment plan; complex view showed nothing
- **Root Cause:**
  1. `useComplexityMode` destructuring was wrong (`{ isComplex }` → should be `{ mode }`)
  2. Groups computed only for activeItems, not for pausedItems/finishedItems
- **Fix:**
  1. Changed to `const { mode } = useComplexityMode(); const isComplex = mode === 'complex'`
  2. Created activeGroups, pausedGroups, finishedGroups computed separately per tab
- **Result:** Groups now compute correctly for all tabs

### Fix 4: Adherence Showing in Wrong Tabs
- **Issue:** Adherence bar showing in paused/finalized tabs (should only show in ativos)
- **Fix:** Conditional rendering: `{showAdherence && <AdherenceBar7d />}` where `showAdherence = activeTab === 'ativos'`
- **Result:** Adherence only visible when active protocols are displayed

### Fix 5: Layout Not Tabular
- **Issue:** Desktop layout still showing cards instead of table rows
- **Root Cause:** ProtocolRow had single flex layout; CSS Grid wasn't distributing cells
- **Fix:** Restructured ProtocolRow with two distinct render paths (variant="card" vs variant="tabular")
  - Tabular: Returns array of 4 independent cell divs for CSS Grid parent
  - Card: Returns single flex container with all info stacked
- **Result:** Desktop now shows proper 4-column table layout with tonal alternation

### Fix 6: Borders Violating Design System
- **Issue:** 1px borders between cells (violated product design strategy)
- **Fix:** Removed all borders; implemented tonal row alternation (surface-container-low for even rows)
- **Result:** Cleaner hierarchy without unnecessary visual noise

### Fix 7: Search Not Opening Edit Form
- **Issue:** Clicking existing medicine in search did nothing
- **Fix:** AnvisaSearchBar now detects if medicine has existing protocol and calls `onEditProtocol(match)`
- **Result:** Search correctly routes to edit form for existing protocols

### Fix 8: Autocomplete Not Title-Cased
- **Issue:** Data from ANVISA comes mixed case; should be normalized
- **Fix:** Applied `toTitleCase()` and `toSentenceCase()` to autocomplete data in AnvisaSearchBar, TreatmentWizard, MedicineForm
- **Result:** All user-facing text is properly formatted

### Fix 9: Case Conversion Strategy (Latest)
- **Issue:** ALL fields were being `toSentenceCase()` (only first word capitalized)
- **Analysis:** Correct approach:
  - Name + Active Ingredient: **Title Case** (capitalize all words)
  - Therapeutic Class: **Sentence Case** (only first word capitalized)
- **Fix:** Updated AnvisaSearchBar, TreatmentWizard, MedicineForm to use correct functions per field
- **Result:** Consistent, semantically correct text formatting

### Fix 10: Stock Calculation Ignoring Dosage Per Intake (Critical)
- **Issue:** 90 comprimidos, 3cp/dia → showing 90 dias (should be 30 dias)
- **Root Cause:** `calculateExpectedDoses()` returns number of DOSES, not PILLS; predictRefill wasn't multiplying by `dosage_per_intake`
- **Fix:** In `refillPredictionService.js`, multiply `expectedDoses * dosagePerIntake` to get daily consumption in pills
  ```javascript
  const expectedDoses = calculateExpectedDoses(activeProtocols, 1)
  const dosagePerIntake = activeProtocols[0]?.dosage_per_intake || 1
  dailyConsumption = expectedDoses * dosagePerIntake
  ```
- **Result:** Stock calculation now accurate for all dosages

## Decisions Made

1. **Case Conversion Strategy:** Domain-aware vs generic
   - Name/Ingredient: Title Case (user-visible medicine names deserve capitalization)
   - Therapeutic Class: Sentence Case (classifier, less emphasis needed)

2. **Stock Calculation:** Multiplied by dosage_per_intake at prediction time
   - Alternative: Change `calculateExpectedDoses()` signature (too invasive, used elsewhere)
   - Chosen: Localized fix in `predictRefill()` where semantic is clearest

3. **Modal vs Inline Edit:**
   - Protocol editing via Modal (ProtocolForm) for focused UX
   - Creation via TreatmentWizard modal for new medicines

4. **Persona Detection:** Medicine count (≤3 vs 7+)
   - Simple = Dona Maria (flat list)
   - Complex = Carlos (grouped table)

## Known Limitations / Future Work

- Stock prediction still uses theoretical calculation when <14 days of logs
  - For Omega-3 with no logs: uses current `dosage_per_intake` only
  - Real data available after 14 days of adherence logging
- Tab animations reset completely (key-based remounting); could optimize with exit animations in future
- Persona detection threshold hardcoded (3 vs 7); could make configurable

## Lessons Learned (For Memory)

- **Framer Motion requires remounting for re-animation** (AP-NNN): Using `key={tabId}` forces cascade to re-trigger
- **Distinguish doses from pills** (R-NNN): `calculateExpectedDoses()` returns doses; must multiply by `dosage_per_intake` for stock calculation
- **Domain-aware case conversion** (R-NNN): Different fields need different strategies (Title Case for names, Sentence Case for classifiers)
- **Product design constraints matter** (R-NNN): No 1px borders; tonal separation > hard lines
- **Data fetching architecture** (R-NNN): TreatmentItem ≠ Protocol schema; edit flows need full protocol via `protocolService.getById()`
