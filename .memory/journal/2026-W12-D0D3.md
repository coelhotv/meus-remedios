# 2026-W12 — Dashboard First Load Performance (D0-D3)

**Data:** 2026-03-20
**Branch:** `perf/dashboard-first-load-d0-d3`
**Baseline:** `bug-logs/first-load_production.har` (39 requests, 1.9s waterfall)

## Entregas

### D0 — Fix lazy loading (vendor-pdf)
- `Dashboard.jsx` importava `ReportGenerator` estaticamente → cadeia transitiva puxava vendor-pdf (589KB) para modulepreload
- Convertido para `React.lazy()` + Suspense; imports no pdfGeneratorService agora são `import()` dinâmico
- **Resultado:** vendor-pdf removido do modulepreload (-172KB gzip no first load)
- **Nota:** feature-stock/history/landing continuam em modulepreload devido a barrel exports em `@shared/services/index.js` → `cachedServices.js` → `stockService`. Refactor de barrel exports seria sprint separado.

### D1+D2 — dailyAdherence client-side (elimina 2 queries)
- `Dashboard.jsx` chamava `adherenceService.getDailyAdherence(7)` em useEffect separado
- Internamente fazia `protocols?select=*` (6.8KB) + `medicine_logs?select=taken_at` (4.2KB)
- Movido para useMemo no DashboardProvider usando dados já disponíveis (protocols + logs 30d)
- **D2 eliminado:** a query simples de protocols (sem joins) vinha do getDailyAdherence, não de outra fonte
- **Resultado:** -2 queries GET, -1 useEffect

### D3 — Cache getCurrentUser
- `getCurrentUser()` não tinha cache (diferente de `getUserId()`)
- Adicionado `_cachedUser` + `_currentUserPromise` com coalescência
- Invalidação no `onAuthStateChange`
- **Resultado:** -2 auth roundtrips (3→1)

## Lições Aprendidas

1. **HAR de localhost ≠ produção**: StrictMode duplica effects, inflando queries 2-3x. Sempre medir em produção.
2. **Barrel exports quebram lazy loading**: `@shared/services/index.js` re-exporta tudo, incluindo stockService. Qualquer import de `@shared/services` puxa a cadeia inteira para o main bundle.
3. **manualChunks ≠ lazy loading**: Vite coloca módulos em chunks separados, mas se há dependência transitiva estática, gera `<link rel="modulepreload">` que anula o benefício.
