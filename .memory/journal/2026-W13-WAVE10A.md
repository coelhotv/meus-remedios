## 🔧 Padrões & Decisões

### Ordem de Declaração (FIX: Temporal Dead Zone)
Problema encontrado: `showFeedback` era usada nos handlers antes de ser declarada → `ReferenceError`

**Solução:** Reordenar conforme padrão obrigatório:
```
States → Hooks → Helper Functions → Handlers → Effects → Render
```

**Resultado:** `showFeedback` declarada após hooks, antes dos handlers que a usam.

### Estrutura de Arquivos

```
src/views/redesign/
  SettingsRedesign.jsx              ← Nova view (independente)

```

## 📝 Lessons Learned

### R-065: Order of Declaration in React Hooks (CONFIRMADO)
- **Pattern:** States → Memos → Effects → Handlers (add: Helper Functions ANTES dos handlers)
- **Why:** Evita Temporal Dead Zone errors quando handlers usam helper functions declaradas depois
- **This Sprint:** `showFeedback` movia para ANTES dos handlers que a usam

### AP-HDZ1: Temporal Dead Zone with useCallback Dependencies (NOVA)
- **Problem:** `showFeedback` declarada em linha 129, usada em `generateTelegramToken` (linha 59)
- **Error:** `ReferenceError: Cannot access 'showFeedback' before initialization`
- **Root Cause:** JavaScript hoisting — `const` não sofre hoisting; precisa estar declarada ANTES do uso
- **Fix:** Mover `showFeedback` para antes dos handlers (logo após hooks)
- **Prevention:** Revisar ordem de declaração em React components antes de commitar
