## 🔧 Padrões & Decisões

### Offline-First + Write-Through
- `emergencyCardService.load()`: localStorage primeiro, fallback Supabase
- `emergencyCardService.save()`: localStorage (síncrono) + Supabase (assíncrono)
- One-time sync no load: se dados em localStorage mas não em Supabase, re-salva

### Idade Calculada
- Input: `birth_date` (DATE via input type="date")
- Compute: useMemo com parseLocalDate (timezone-aware)
- Validação: Zod schema `z.string().date()`

### Validação Zod
- `display_name`: obrigatório, 2-200 chars, trimmed
- `birth_date`, `city`, `state`: opcionais, .nullable().optional()
- Brasileirosus states: const BRAZILIAN_STATES = ['AC', 'AL', ..., 'TO']

### Layout Hub
- Não é sidebar + sections (Wave 9)
- É view centralizada com 3 seções: Header + Cards + Tools
- Responsive: desktop grid 2x1 para cards, mobile stack 1x1

## 📝 Lessons Learned

### R-123: Profile Hub as Centralized Entry Point
- **Pattern:** Single hub view with Header + Cards + Tools
- **Why:** Melhor UX: "Quem sou eu?" centralizado vs. sidebar + seções dispersas
- **Application:** Próximas views (dashboard, stock) podem seguir hub pattern

### R-124: One-Time Sync localStorage → Supabase
- **Pattern:** No load, se source === 'local' E DB column NULL, re-save via service
- **Why:** Corrige dívida técnica sem refactoring do service (write-through já funciona)
- **Application:** Aplica a qualquer coluna que deveria estar em Supabase desde o início

### AP-HDZ2: Temporal Dead Zone com Componentes Internos
- **Problem:** ToolCard function definida DEPOIS do ToolCard usage (no render)
- **Fix:** Definir ToolCard ANTES (ou mover para arquivo separado)
- **Prevention:** Revisar ordem de declaração em componentes antes de submitir
