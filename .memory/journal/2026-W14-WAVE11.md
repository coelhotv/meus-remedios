## Decisões

- **z-index Modal hardcoded em 1200**: dois arquivos de tokens CSS (`src/styles/` e `src/shared/styles/`) definem `--z-modal-overlay` com valores diferentes; o último carregado vencia e sobrescrevia para 401. Solução: hardcode direto no Modal.css com comentário explicativo.
- **Botão X à esquerda**: reordenado no JSX + `justify-content: flex-start` + `flex: 1` no h2. Consistência mobile (iOS pattern).
- **ExportDialog segmented control**: mesma implementação do LogForm — `format-toggle` + `format-toggle-btn` com `.active`.

## Lições Aprendidas

- **AP-W24 (BUG ABERTO)**: FABs e chatbot trigger ainda aparecem sobre modais mesmo após correção de z-index. Hipótese: `transform: translateX(-50%)` no `.doseFab` cria stacking context que isola o elemento. Investigar em Wave 12.
- **Dual token files**: o projeto tem DOIS arquivos de tokens z-index. Ao adicionar tokens de layering, atualizar AMBOS: `src/styles/tokens/z-index.css` e `src/shared/styles/tokens/z-index.css`.
- **ESLint + destructuring alias em JSX**: `icon: FormatIcon` em parâmetro de map não é reconhecido como "usado" pelo `no-unused-vars` do projeto. Usar `const OptionIcon = option.icon` dentro do corpo da função resolve.
- **overscroll-behavior: contain**: fix correto para bounce/rubber band lateral em modais iOS. Aplicar em `.modal-content`.
- **Validação em celular real**: essencial para detectar comportamentos iOS específicos (bounce, z-index, bottom sheet). `vite.config.js server.host: true` simplifica o fluxo.
