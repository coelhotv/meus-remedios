### UX: Redução de ruído visual no StockCard (PR #402)

**Problema:** StockCard exibia todas as entradas de estoque com o mesmo peso visual:
1. Lotes com `quantity = 0` (zerados por FIFO via `stockService.decrease()`)
2. Entradas geradas automaticamente por `logService.delete()` → `stockService.increase(id, qty, "Dose excluída (ID: X)")`

**Fix:** Classificação das entradas em 3 tipos antes de renderizar:
- **Lotes ativos**: `quantity > 0` e `notes` não é ajuste de sistema → exibidos normalmente
- **Lotes consumidos**: `quantity === 0` → ocultos por padrão (seção colapsável)
- **Ajustes automáticos**: `notes` começa com `"Dose excluída"` ou `"Ajuste de dose"` → ocultos por padrão

**Padrão aplicado:**
```javascript
const SYSTEM_NOTE_PREFIXES = ['Dose excluída', 'Ajuste de dose']
const isSystemAdjustment = (e) => SYSTEM_NOTE_PREFIXES.some(p => e.notes?.startsWith(p))
const { activeEntries, consumedEntries, systemEntries } = stockEntries.reduce(...)
```

**Correções adicionais do review Gemini:**
- `new Date()` → `parseLocalDate()` em formatDate/isExpired/isExpiringSoon (violação R-020)
- 3x `Array.filter()` → uma passagem com `reduce()` (performance)
- `||` → `Array.some()` + constante de módulo (extensibilidade)

## Lições Aprendidas

- **Mock Supabase por tabela precisa cobrir TODOS os métodos chamados:** quando adicionamos query a nova tabela (`medicine_logs` com `.gte()`), mocks de teste antigos quebram. Sempre revisar os mocks ao adicionar queries.
- **localStorage em jsdom pode não funcionar:** mesmo com `environment: 'jsdom'`, se `--localstorage-file` tiver path inválido, localStorage não está disponível. Usar `vi.stubGlobal('localStorage', mockObj)` é mais confiável.
- **Option A para bot (duplicar lógica):** refillPredictionService é client-side mas bot é server-side. Duplicar a lógica de consumo real no bot é mais limpo que compartilhar código entre client/server com path aliases diferentes.
