## Decisões

- **ConfirmDialog vs window.confirm**: Modal reutilizável dá controle total sobre styling + acessibilidade
- **Presenter pattern (12.3+12.4)**: Separation de concerns — container (data) vs presenter (visual)
- **Framer Motion footer**: spring(stiffness: 300) combina physics realism + responsividade
- **Print media**: #e8f5f3 hardcoded (Verde Saúde light) para legibilidade em impressão
- **Glassmorphism footer**: backdrop-blur 12px + 0.85 opacity para legibilidade sem ocluir conteúdo

---

## Lições Aprendidas

- **AP-W25: Modal branching em App.jsx é pattern estabelecido** — todos os redesign views seguem isRedesignEnabled ? NewView : OldView
- **Modal imports**: MedicineForm já está W11-redesigned via CSS (não precisa nova versão)
- **Service discovery**: getConsultationData já existe (services/consultationDataService) — não duplicar
- **Dashboard context**: useDashboardContext() centraliza dados (não chamar services direto)
- **Footer z-index**: fixed 100 não conflita com modal 1200 (layer de BottomNav é 64, mas footer é acima dela)
