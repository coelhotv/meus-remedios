### 2026-02-26 01:00 — Priority Mapping Bug Fix (PR #204) [BUG-FIX]
**Type:** BUG-FIX
**Files:** `api/gemini-reviews/persist.js`, `api/gemini-reviews/create-issues.js`, `.github/scripts/create-issues.cjs`, `.github/scripts/persist-reviews.cjs`
**Status:** [GRADUATED]

**Context:**
Reviews with priority `HIGH` were incorrectly generating `[Medium]` GitHub issues instead of `[High]`. The bug was caused by two issues:

1. **Field mismatch**: API used `severity` field but code expected `priority`
2. **Logic bug**: Only fetched `media` (Portuguese for "medium") from the database, ignoring other priorities

**Root Cause:**
```javascript
// Bug: Only fetching 'media' (medium)
const { data: existing } = await supabase
  .from('gemini_reviews')
  .select('id, priority')
  .eq('priority', 'media')  // ❌ Only 'media'
```

**Fix Applied:**
1. Updated API to handle both `severity` and `priority` fields
2. Updated GitHub scripts to fetch all priorities, not just `media`
3. Added proper mapping: `high` → `[High]`, `medium` → `[Medium]`, `low` → `[Low]`, `critical` → `[Critical]`

**PRs:**
- #204: Priority mapping fix merged
- #186: Test validation PR closed
- Branch `test/gemini-priority-validation` deleted
- 18 test issues closed

### 2026-02-22 18:56 — Agent Coordination Failure & Recovery [CRITICAL]
**Type:** PROCESS
**Files:** N/A (process lesson)
**Status:** [ACTIVE]

**Context:**
Coordination of Sprint 5 and start of Sprint 6 for Workflow Intelligence. Attempted to accelerate delivery by delegating coding tasks without rigorous PR review process.

**What Went Wrong:**
1. **Tried to accelerate the process** — Delegated coding tasks without requiring rigorous PR reviews
2. **Didn't follow code review process** — Code agents were merging their own PRs
3. **Prioritized speed over quality** — Created sequential tasks without pauses for review
4. **Lack of supervision** — Didn't verify if agents were following the mandatory workflow

**Course Corrections Implemented:**
1. **Rigorous review process**:
   - Code creates PR → DOES NOT merge
   - Debug/Code reviews → Reads all Gemini Code Assist comments
   - Code applies CRITICAL/HIGH fixes
   - Code creates issues for MEDIUM/LOW
   - DevOps merges → Only after explicit approval

2. **Mandatory checklist between tasks**:
   - PR reviewed by Gemini Code Assist?
   - CRITICAL/HIGH issues resolved?
   - Atomic commits made?
   - Validations (lint, tests) passed?
   - Only then: merge

3. **Separation of responsibilities**:
   - Code: Implements, fixes
   - Debug: Reviews, validates
   - DevOps: Merges, cleans branches
   - Architect: Plans, documents

**What Worked (After Corrections):**
1. **PR #124 (Migration SQL)**:
   - Code created migration
   - Debug reviewed, found CRITICAL issues
   - Code fixed (3 commits)
   - Backlog issues created (#125-130)
   - DevOps merged
   - Result: Quality code in production

2. **Refactoring documentation**:
   - Architect created complete document (2,377 lines)
   - Clear specification for next sprints
   - Mapping of what changes/continues/is removed

**Rule Candidates:**
- **RULE:** Never delegate merge to the agent that created the PR → R-060
- **RULE:** Always have a pause for review between coding and next task → R-061
- **RULE:** Quality > Speed — One well-reviewed PR is worth more than 3 rushed PRs → R-062
- **RULE:** Process is protection — The rigorous workflow exists to prevent errors → R-063
- **ANTI-PATTERN:** Code agent merging its own PR without external review → AP-020
- **ANTI-PATTERN:** Skipping Gemini Code Assist review to save time → AP-021
- **ANTI-PATTERN:** Sequential task creation without validation gates → AP-022
