## 📝 Lessons Learned

**R-136 (New):** When refactoring for batch operations (onRegisterAll), ensure the callback implementation matches the UI promise (all items selected = all items registered). The UI is the contract.

**R-137 (New):** Responsive layout with desktop-specific column ratios (1fr 2fr) should use CSS media queries, NOT inline styles. Confirmed by Gemini Code Assist as best practice.

**R-138 (New):** Decline refactoring suggestions that are out-of-scope for the current wave, but document them in GitHub backlog issues with full context. Future sprints can prioritize based on impact.

**AP-W21 (New):** Callback handlers that promise batch operations (register all doses) but only implement single items (doses[0]) create UX contradictions. Test batch operations explicitly.
