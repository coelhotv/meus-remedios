## Decisões Técnicas

- **reminderSuggestionData via useMemo** (não useState+useEffect): evita react-hooks/set-state-in-effect. `isSuggestionDismissed` e `analyzeReminderTiming` são funções estáveis importadas → eslint-disable localizado.
- **dismissedSuggestionId** (1 state) substitui 3 states: reminderSuggestion + suggestionProtocolId + suggestionProtocolName.
- **Zero novas chamadas Supabase**: toda computação sobre cache SWR existente do DashboardProvider.
- **Sub-agentes haiku** em fase 1 paralela (4 agentes × 2 componentes cada) — economia de tokens com supervisão Sonnet.

## Lições Aprendidas

- `react-hooks/set-state-in-effect`: chamar setState dentro de useEffect body é flagado → converter para useMemo que computa o valor derivado diretamente.
- `react-hooks/preserve-manual-memoization`: funções estáveis importadas (não closures) que não estão em deps array → eslint-disable com comentário explicativo é a solução correta.
- Parâmetro não utilizado `_newTime` em useCallback ainda é flagado pela regra no-unused-vars — remover o parâmetro completamente é mais limpo.
- Mock HTML/CSS de alta fidelidade antes de 3+ componentes visuais novos = ROI enorme (evita retrabalho pós-validação visual).
- Worktree isolation com sub-agentes: SDK auto-limpa worktrees quando commits vão para o mesmo branch — não é problema.
