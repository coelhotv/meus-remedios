## 📚 Rules & Learnings

### New Rules Added

- **R-137:** Typography weights must be ≥400 for accessibility (no light/thin fonts for elderly patients)
- **R-138:** lucide-react icons always paired with text labels (accessibility + UX clarity)
- **R-139:** CSS `@import` for fonts in token files, never `<link>` in index.html (scoped delivery)
- **R-140:** Heading specific weights via `--heading-N-weight` variables (level-appropriate hierarchy)

### Anti-Patterns Reinforced

- **AP-D01:** Using font weights <400 → breaks WCAG 2.1 AA for geriatric users
- **AP-D02:** Adding fonts via `<link>` in index.html → affects all users (breaks gradual rollout)
- **AP-D03:** Icon-only UI without text label → accessibility failure + UX confusion

---

## 🎓 Lessons Learned

### Challenge 1: Backward Compatibility vs. New Design
- **Problem:** Spec required size aliases (`--text-xs` through `--text-5xl`) for backward compat
- **Learning:** Token systems must layer (semantic names + numeric aliases) to avoid breaking existing components
- **Applied:** All tokens added both ways (display/headline/title AND text-4xl/3xl/2xl)

### Challenge 2: Form Element Inheritance
- **Problem:** Weren't sure if inputs/buttons would inherit font from container
- **Learning:** CSS `font-family` inherits automatically; explicit selector for form elements ensures clarity
- **Applied:** Added explicit `[data-redesign="true"] input, textarea, select, button` rules

### Challenge 3: Typography Scale Completeness
- **Problem:** Missing heading defaults that could cause h1-h6 visual inconsistency
- **Learning:** Define ALL heading variables upfront (size + weight + line-height + tracking per level)
- **Applied:** 6 heading levels × 2-3 properties each = comprehensive hierarchy

## 📚 New Rules & Anti-Patterns

### New Rules Added
- **R-118:** Redesign CSS colors must use CSS variables + color-mix() (AP-024 prevention)
- **R-119:** Size properties (.btn-sm/md/lg) only in size classes, never in variants (AP-W02 prevention)
- **R-120:** Consolidate CSS rules via selector grouping when >80% duplication (AP-C02 prevention)

### New Anti-Patterns Documented
- **AP-024:** Hardcoded colors bypass design system → future maintenance burden
- **AP-W02:** Variant-level min-height overrides size class functionality
- **AP-C02:** CSS rule duplication without consolidation → cascading updates needed

## 🎓 Key Learnings

### Challenge 1: Not Reading Memory Files Before Coding
- **Problem:** Delivered Wave 3 without consulting `.memory/rules.md` + `.memory/anti-patterns.md`
- **Result:** 8+ reviewer issues (all preventable via R-065 + AP-023)
- **Learning:** ALWAYS read memory files FIRST (R-065 is non-negotiable)
- **Applied:** Created AP-024 to document this specific error pattern

### Challenge 2: Hardcoded Colors in Token-Based Systems
- **Problem:** Used hex/rgba directly in CSS instead of checking tokens.redesign.css
- **Result:** Design system bypass, harder theme updates, inconsistency
- **Learning:** New rule R-118 ensures all colors use --color-* variables + color-mix()
- **Applied:** Gemini Code Assist flagged all 6 hardcoded color instances → all fixed

### Challenge 3: Button Size Class Overrides
- **Problem:** `.btn-primary` had `min-height: 56px` → overrode `.btn-sm/.btn-md`
- **Result:** API contract broken (size prop didn't work on primary buttons)
- **Learning:** Size properties belong in size classes ONLY, never in variants (R-119)
- **Applied:** Removed min-height from `.btn-primary`, preserved in `.btn-lg` override

### Challenge 4: CSS Rule Duplication Without Consolidation
- **Problem:** `.btn-secondary` and `.btn-outline` were 100% identical
- **Result:** 36 lines of duplicated CSS → harder maintenance
- **Learning:** Use selector grouping (`,`) for shared base rules, separate size/state modifiers (R-120)
- **Applied:** Reduced button + list-item rules by consolidating shared properties
