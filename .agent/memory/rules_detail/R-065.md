# R-065 — Read Memory Before Coding [CRITICAL]

## Rule
Every code agent MUST read `.memory/rules.md` and `.memory/anti-patterns.md` BEFORE writing any code.

**Required sequence:**
1. Read `.memory/rules.md` → note relevant R-NNN rules
2. Read `.memory/anti-patterns.md` → note relevant AP-NNN patterns
3. Check for duplicate files (R-001)
4. Only then start coding

## Why
PR #132 review process. Agent repeated a known mistake (from AP-023) that was already documented. Reading memory first takes ~2 minutes and prevents repeating costly incidents.

## DEVFLOW note
In DEVFLOW this rule is enforced architecturally via the mandatory Bootstrap phase. This rule documents the intent for legacy `.memory/` context.
