# R-164: Supabase auth persistence em React Native — getSession + initialRouteName dinâmico

**Criado em:** 2026-04-12
**Contexto:** React Native / Expo com Supabase Auth e @react-navigation

---

## A Regra

O componente de navegação raiz DEVE restaurar a sessão Supabase assincronamente no mount
e usar três estados para controlar o `initialRouteName`. SecureStore chunked é pré-requisito
(R-160/AP-H07) mas não suficiente sem esta lógica de navegação.

---

## Por Que Esta Regra Existe

O Supabase restaura sessão do storage de forma assíncrona. Se o Navigator for montado
com `initialRouteName` fixo (ex: sempre "Smoke"), o utilizador é sempre enviado para
o ecrã de login mesmo que exista uma sessão válida guardada.

**Resultado sem esta regra:** Utilizador faz login, fecha app, reabre → pede login de novo.

---

## Padrão de Implementação

```jsx
// Navigation.jsx
import { useEffect, useState } from 'react'
import { View, ActivityIndicator } from 'react-native'
import { NavigationContainer } from '@react-navigation/native'
import { createNativeStackNavigator } from '@react-navigation/native-stack'
import { supabase } from '../platform/supabase/nativeSupabaseClient'

const Stack = createNativeStackNavigator()

export default function Navigation() {
  // CRÍTICO: undefined (não null) = a verificar
  const [session, setSession] = useState(undefined)

  useEffect(() => {
    // 1. Restaurar sessão persistida no SecureStore
    supabase.auth.getSession().then(({ data: { session: s } }) => {
      setSession(s ?? null)
    })

    // 2. Actualizar em tempo real (login/logout durante uso)
    const { data: { subscription } } = supabase.auth.onAuthStateChange((_event, s) => {
      setSession(s ?? null)
    })

    // 3. Cleanup obrigatório
    return () => subscription.unsubscribe()
  }, [])

  // 4. Spinner enquanto verifica — NUNCA mostrar tela errada
  if (session === undefined) {
    return (
      <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
        <ActivityIndicator size="large" color="#2563eb" />
      </View>
    )
  }

  // 5. initialRouteName dinâmico
  return (
    <NavigationContainer>
      <Stack.Navigator initialRouteName={session ? ROUTES.HOME : ROUTES.SMOKE}>
        {/* screens */}
      </Stack.Navigator>
    </NavigationContainer>
  )
}
```

---

## Os Três Estados

| Valor | Significado | Acção |
|-------|-------------|-------|
| `undefined` | A verificar sessão | Mostrar spinner |
| `null` | Sem sessão | `initialRouteName = SMOKE` |
| `object` | Sessão activa | `initialRouteName = HOME` |

**Porquê `undefined` e não `null`?**
`null` seria indistinguível de "sem sessão confirmada" → flash de login antes de verificar.
`undefined` = estado transitório que só existe durante a verificação inicial.

---

## Pré-requisitos

1. **SecureStore chunked** ([AP-H10](../anti-patterns_detail/AP-H10.md), R-160) — tokens Supabase > 2048 bytes precisam de chunking
2. **`nativeSupabaseClient`** configurado com `secureStoreAuthStorage` como storage
3. **`@react-navigation/native`** e `@react-navigation/native-stack` instalados

---

## Fluxo Completo

```
App abre
  → Navigation monta → session = undefined → spinner
  → getSession() → SecureStore.getItem (chunked)
  → token válido → Supabase valida → session = {user, access_token, ...}
  → session !== undefined → NavigationContainer monta
  → initialRouteName = HOME → utilizador vê HomeScreen ✅

App abre (sem sessão)
  → Navigation monta → session = undefined → spinner
  → getSession() → SecureStore.getItem → null
  → session = null → NavigationContainer monta
  → initialRouteName = SMOKE → utilizador vê SmokeScreen → Login ✅

Utilizador faz logout
  → onAuthStateChange callback → session = null
  → NavigationContainer re-renderiza → vai para SMOKE ✅
```
