# R-025 — Fallback Patient Identity From Email Handle [MEDIUM]

## Rule
If the profile has no display name, derive the PDF patient label from the **email local-part** (before the @) before falling back to `"Paciente"`.

## Why
PR #421 PDF consultation patch (2026-03-24). "Paciente" is clinically useless. Email handle is a reasonable proxy until the database has a dedicated name field.

## Priority order
1. `profile.display_name` (if set)
2. `email.split('@')[0]` (email local-part)
3. `"Paciente"` (absolute fallback)
