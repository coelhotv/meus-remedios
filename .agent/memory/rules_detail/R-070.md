# R-070 — No setTimeout in act() Blocks [HIGH]

## Rule
Never use `setTimeout(resolve, N)` inside `act()` to "wait for component to load".

Use `waitFor(() => expect(result.current.isLoading).toBe(false))` instead.

## Why
useDashboardContext.test.jsx had 11 hanging instances. Hardcoded delays fail when the component takes longer than N milliseconds (flaky in CI, especially low-RAM machines).

## When to use waitFor vs fake timers
- **waitFor** — for React state changes (loading flags, data arrival) → R-070
- **vi.useFakeTimers** — for timer control in code under test (debounces, delays) → R-073

## Example
```javascript
// WRONG — hangs if component takes >100ms
await act(async () => {
  await new Promise(resolve => setTimeout(resolve, 100))
})

// CORRECT — polls until condition or timeout
await waitFor(() => expect(result.current.isLoading).toBe(false))
```
