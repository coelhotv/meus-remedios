# R-073 — Use Fake Timers for Timer-Dependent Tests [MEDIUM]

## Rule
Tests depending on real elapsed time are inherently flaky in CI. Use `vi.useFakeTimers()` and `vi.runAllTimersAsync()` for deterministic control.

Always call `vi.useRealTimers()` in `afterEach` or `finally`.

## When to use
- Timer control in code under test (debounces, throttles, delays) → use R-073 (fake timers)
- Waiting on React state changes → use R-070 (waitFor)

## Pattern
```javascript
beforeEach(() => { vi.useFakeTimers() })
afterEach(() => { vi.useRealTimers() })

test('debounce fires after 300ms', async () => {
  trigger()
  await vi.runAllTimersAsync()
  expect(handler).toHaveBeenCalledOnce()
})
```
