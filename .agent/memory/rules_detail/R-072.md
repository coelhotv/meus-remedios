# R-072 — Always Resolve Dangling Promises [HIGH]

## Rule
When a test creates a manually-controlled Promise (capturing `resolve` from the executor), ALWAYS resolve it in a `finally` block.

If an assertion fails before the manual `resolve()` call, the Promise stays pending and Vitest hangs.

## Why
StockForm.test.jsx "disable buttons while submitting" test. An assertion failure left `resolveSave()` uncalled, hanging the entire test suite.

## Pattern
```javascript
// WRONG — resolveSave() never called if waitFor throws
await waitFor(() => { expect(...).toBeInTheDocument() })
resolveSave()

// CORRECT
try {
  await waitFor(() => { expect(...).toBeInTheDocument() })
} finally {
  resolveSave?.()
}
```
