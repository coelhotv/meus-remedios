# R-191 — Partition Doses into Semantic Blocks

## Rule

Antes de disparar notificações de doses para o usuário, particionar o array de doses em blocos semânticos usando `partitionDoses()`:
- **by_plan**: doses do mesmo treatment_plan (bulk action)
- **misc**: doses sem protocolo (não agrupáveis)
- **individual**: doses isoladas (1 dose, nenhum padrão)

Disparar **1 push por bloco**, nunca 1 push por protocolo.

## Why

- **Lock Screen Overload**: Usuários com 8+ protocolos recebem 8 notificações idênticas com timestamp diferentes
- **UX Degradation**: Notificação loss (iOS caulks duplicates), confusão visual no lock screen, maior taxa de dismiss sem ler
- **Pattern Discovered W25**: Wave N1 planning identificou disparos paralelos em `checkRemindersViaDispatcher` que criava N × protocolos notificações por minuto

## How To Apply

1. **Backend (server/bot/tasks.js)**:
   ```js
   import { partitionDoses } from './utils/partitionDoses'
   const blocks = partitionDoses(dosesForNotification)
   for (const block of blocks) {
     await dispatchNotification(userId, { ...block, kind: block.type })
   }
   ```

2. **API/Notify.js**: Respeitar `dispatchInputSchema.kind` enum (dose_reminder_by_plan / dose_reminder_misc / dose_reminder_individual)

3. **Telegram Channel**: Enviar 1 mensagem por bloco com botões inline numerados (callback_data < 64 bytes)

4. **Expo Push Channel**: Enriquecer payload com `notificationLogId`, `blockedDoseIds`, dedup via `shouldSendGroupedNotification`
