# R-194 — Cold Start Processing Guard (useRef)

## Rule

Em hooks que processam `getLastNotificationResponseAsync()` na montagem, usar um `useRef` flag para guardar contra reprocessamento:

```js
const coldStartProcessed = useRef(false)

useEffect(() => {
  if (coldStartProcessed.current) return
  
  const response = await getLastNotificationResponseAsync()
  if (response?.notification) {
    navigate(...)
    coldStartProcessed.current = true
  }
}, []) // deps = [] (só roda em mount)
```

## Why

- **AP-116 Prevention**: `useEffect([session])` + `getLastNotificationResponseAsync` reexecuta quando session muda (logout+login)
- **Double Navigation**: Sem guard, navega 2x na mesma notificação → visual stutter / incorrect screen state
- **Cold Start Contract**: Garantir que cold start é processado **exatamente uma vez**

## How To Apply

Ao implementar usePushNotifications:
1. Adicionar `useRef(false)` no início
2. Check `coldStartProcessed.current` antes de processar
3. Setar flag após sucesso
4. Usar empty deps array `[]` para garantir efeito dispara só na montagem
