# R-170: Bypass Pty Host via .sh scripts

> **Contexto:** Instabilidade do Pty Host no Antigravity/VS Code ao processar comandos longos com múltiplos pipes e JSON parses (ex: gh api).
> **Solução:** Encapsular a lógica em um script .sh temporário e executá-lo.

## 📝 Descrição

Comandos complexos do GitHub CLI (`gh`) integrados com `jq` e múltiplos redirecionamentos podem causar travamento do terminal do agente devido à forma como o Pty Host gerencia o buffer de saída e entrada.

### ❌ Anti-padrão (Causa travamento)
```bash
gh api repos/OWNER/REPO/comments | jq -r '.[] | select(...)' | xargs -I {} gh api ... --input -
```

### ✅ Solução (R-170)
1. Criar um arquivo temporário (ex: `/tmp/reply.sh`).
2. Escrever o comando complexo no arquivo.
3. Executar o arquivo: `bash /tmp/reply.sh`.
4. Remover o arquivo após a execução.

## ⚙️ Exemplo de Implementação

```bash
cat <<' INNER_EOF' > /tmp/gh_reply.sh
# Comandos complexos aqui
gh api ... | jq ... | gh api ...
INNER_EOF
bash /tmp/gh_reply.sh
rm /tmp/gh_reply.sh
```
