# R-001 — Duplicate File Check [CRITICAL]

## Rule
Before modifying ANY file, search for duplicates with the same name across the project. If duplicates exist, trace which file is actually imported. Modify only the canonical file; delete duplicates.

## Why
5+ incidents in W06-W08 where the wrong duplicate was modified, causing production bugs that were hard to trace. The project has had component files split across features with identical names.

## Commands
```bash
find src -name "*TargetFile*" -type f
grep -r "from.*TargetFile" src/ | head -20
```

## Related anti-pattern
AP-001 (if exists) — Wrong File Modified

## Notes
This rule applies before EVERY file modification, no exceptions.
