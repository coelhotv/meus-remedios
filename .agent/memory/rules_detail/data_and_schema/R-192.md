# R-192 — Supabase .contains() for Array Columns

## Rule

Quando filtrar colunas array no Supabase (ex: `protocol_ids`), usar `.contains()` em vez de fetch-all + filter em memória:

```js
// ❌ ERRADO
const { data: records } = await supabase.from('table').select()
const filtered = records.filter(r => r.protocol_ids.includes(protocolId))

// ✅ CORRETO
const { data: records } = await supabase
  .from('table')
  .select()
  .contains('protocol_ids', [protocolId])
```

## Why

- **Query Pushdown**: Database filtra linhas antes de transmitir — reduz payload HTTP
- **Escalabilidade**: Com N=10k records e filtro resultando em 10, transferir 10k records em vez de 10 é 1000x mais lento
- **Wave N1.2**: Otimização crítica para Sprint N1.2 (callback handlers com busca de protocolos aplicáveis a múltiplas doses)

## How To Apply

Sempre que um WHERE clause envolve verificação de membership em array:
1. Converter para `.contains()` no RLS/query
2. Validar que a coluna é array no schema
3. Testar em Supabase dashboard com EXPLAIN ANALYZE
