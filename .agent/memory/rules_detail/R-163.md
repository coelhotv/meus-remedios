# R-163: Expo Go SDK 53 — usar AppRegistry directamente, não registerRootComponent

**Criado em:** 2026-04-12
**Contexto:** Projectos Expo sem expo-router, usando Expo Go SDK 53

---

## A Regra

Em projectos Expo que **não usam expo-router**, o entrypoint `index.js` deve usar
`AppRegistry.registerComponent` directamente do `react-native`, não `registerRootComponent` do `expo`.

---

## Por Que Esta Regra Existe

`registerRootComponent` do Expo SDK 53 tem detecção automática de expo-router através de:

1. **Parâmetro `transform.routerRoot=app`** no URL de carregamento do bundle (enviado pelo Expo Go)
2. **Presença do directório `src/app/`** no projecto

Quando detecta qualquer um destes, tenta importar `expo-router` que se não estiver instalado
causa crash: `TypeError: Cannot read property 'get' of undefined`

A mensagem de erro é enganadora — não menciona "expo-router" e a stack trace aponta para
`NativeModules`, tornando o diagnóstico muito difícil.

---

## Como Aplicar

### index.js — Padrão Correcto

```js
// index.js — Entrypoint do Expo
// Polyfills globais PRIMEIRO — antes de qualquer lib que dependa deles
import './polyfills'
// AppRegistry direto do RN: bypass do registerRootComponent do Expo SDK 53
// O registerRootComponent detecta transform.routerRoot=app do Expo Go e tenta
// inicializar expo-router — que não está instalado — causando crash (AP-H09)
import { AppRegistry } from 'react-native'
import App from './App'

AppRegistry.registerComponent('main', () => App)
```

### index.js — Padrão ERRADO (não usar)

```js
// ERRADO ❌ — crasha em Expo Go SDK 53 sem expo-router
import { registerRootComponent } from 'expo'
import App from './App'
registerRootComponent(App)
```

---

## Directórios Reservados

Não usar estes nomes mesmo que não uses expo-router:

| Directório | Problema |
|-----------|---------|
| `src/app/` | Detectado como root do expo-router |
| `app/` (na raiz) | Idem |

**Nomes seguros para navegação:** `src/navigation/`, `src/screens/`, `src/router/`

---

## Quando Usar `registerRootComponent`

Só usar `registerRootComponent` quando:
- O projecto TEM expo-router instalado E configurado
- Ou quando se usa Expo SDK < 50 (sem detecção automática)
