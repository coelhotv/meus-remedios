# ADR-004: Structured Logging for Telegram Bot Debugging

**Status:** Accepted | **Date:** 2026-03-20 | **Source:** PR #411
**Tags:** chatbot, telegram, logging, debugging

## Context
Telegram bot debugging was difficult due to unstructured logs mixed with console output.

## Decision
Implement structured JSON logging (JSON Lines format) for Telegram bot with contextual information.

## Implementation
- Log events as JSON objects (one per line)
- Include: timestamp, event_type, user_id, context, error_details
- Write to file: `.logs/telegram-bot.jsonl`
- Parse with `jq` or log aggregation tool

## Example Log
```json
{"ts":"2026-04-08T10:30:45Z","event":"message_received","user_id":"12345","text":"help","error":null}
{"ts":"2026-04-08T10:30:46Z","event":"response_sent","user_id":"12345","tokens":250,"latency_ms":1200}
```

## Consequences
✅ Easy to parse and filter logs
✅ Integrates with log aggregation services
⚠️ Requires log rotation strategy
⚠️ More verbose than unstructured logs
