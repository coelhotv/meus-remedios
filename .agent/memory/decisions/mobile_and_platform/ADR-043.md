# [ADR-043] Hardening G1→G2→G3 em Fase 1 CRUD (parity test + decisão pré-G3 + PR atômica)

## Contexto
Fase 1 do projeto **Evolução CRUD Native** introduz os gates G1 (Copy) → G2 (Extract) → G3 (Migrate) pela primeira vez em produção. Risco maior: G3 swap do service web pode causar regressão silenciosa em produção.

Estágio atual: closed alpha, time interno apenas. Validação E2E pré-merge feita por PO em dev local.

## Decisão
Aplicar 4 itens de hardening na spec Fase 1; descartar 2 itens (over engineering para o estágio atual).

### Aplicados
1. **G2 ganha critério "suite web roda contra factory"** — testes existentes de `medicineService.test.js` web rodam contra `createMedicineRepository` injetado com mock client. Tudo verde = G2 passa.
2. **Parity test snapshot por método** (`createMedicineRepository.parity.test.js`) — output factory ≡ output service local com mesma fixture, modulo timestamps.
3. **Decisão pré-G3 sobre `costAnalysisService`/`avg_price`** — antes de M3.1. Recomendação: opção B (hook `useMedicines` web absorve cálculo, factory fica pura).
4. **PR G3 atômica** — somente swap web, sem refactor adicional, sem outro domínio.
5. **Smoke checklist E2E PO** em `docs/operations/MEDICINES_G3_SMOKE_CHECKLIST.md`, marcado no PR antes do merge.

### Descartados (mas anotados para reavaliação pré-beta externo)
- **Feature flag de swap web** — overhead vite config + provider + env var não compensa para closed alpha. Rollback = `git revert` + redeploy Vercel ~3min.
- **Playbook formal de incidente** — prematuro. Time interno, blast radius pequeno, comunicação ad-hoc suficiente.

## Justificativa
Closed alpha + time interno + PO valida E2E = blast radius muito menor que cenário B2C. Gates G1/G2 com hardening + PR atômica + smoke checklist cobrem ~90% do risco real sem custo desproporcional.

Hardening complementar (flag, playbook) será reavaliado quando o produto entrar em beta externo.

## Consequências
- G2: gate mais rigoroso, +1 arquivo de teste (`parity`).
- G3: gate mais documentado, +1 doc (`SMOKE_CHECKLIST.md`).
- Sem fricção operacional adicional para closed alpha.
- Confiança recalibrada: G1 alta · G2 alta · G3 média-alta. Aceitável p/ estágio atual.

## Aplicação
- `plans/backlog-native_app/EXEC_SPEC_FASE1_MEDICAMENTOS.md` — seção Quality Gates + Decisão pré-G3.
- `docs/operations/MEDICINES_G3_SMOKE_CHECKLIST.md` — checklist novo.
- Reaplica em Fases 2 (Protocolos) e 3 (Estoque) com adaptações.
