# ADR-017: Profile Hub: Centralized User Data Management

**Status:** Accepted | **Date:** 2026-03-28 | **Source:** PR #436
**Tags:** ux, profile, data

## Context
User profile information was scattered across Settings, Dashboard, and Health History, making it hard to manage account data.

## Decision
Create unified **Profile Hub** as central location for:
- Account settings
- Health information
- Emergency contacts
- Data preferences
- Privacy settings

## Implementation
- Single feature module: `src/features/profile/`
- Sub-routes: /profile/account, /profile/health, /profile/emergency, etc.
- Unified navigation

## Consequences
✅ Single source of truth for profile
✅ Better UX for data management
✅ Easier maintenance
⚠️ Requires data migration
⚠️ Navigation changes needed
