# ADR-009: CSS Scoped Design Token System

**Status:** Accepted | **Date:** 2026-03-24 | **Source:** PR #416
**Tags:** css, design-tokens, architecture

## Context
Need ability to run both old (neon) and new (healthcare) design systems in parallel during migration.

## Decision
All redesign tokens scoped under CSS attribute selector `[data-redesign='true']`, allowing coexistence of styles.

## Implementation
```css
[data-redesign="true"] {
  --color-primary: #006a5e;
  --color-secondary: #005db6;
  /* ... new tokens ... */
}

[data-redesign="false"],
:not([data-redesign]) {
  --color-primary: #00ff00;
  /* ... old tokens ... */
}
```

## Consequences
✅ Complete style isolation
✅ Easy to toggle designs
✅ No CSS conflicts
⚠️ CSS file larger (both systems shipped)
⚠️ Requires careful specificity management
