# ADR-032 — Executar Fase 7: Migração Web para apps/web

## Status
**PROPOSED** — aguardando aprovação humana explícita

## Contexto

ADR-026 estabeleceu que a web permanece na raiz durante as Fases 0-6, e que a Fase 7 (move para `apps/web`) só pode ser iniciada mediante aprovação explícita do maintainer.

As pré-condições documentadas na spec foram verificadas em 2026-04-19:

| Condição | Status |
|----------|--------|
| Fases 1–6 concluídas | ✅ Sprint 6.5 `completed` |
| Shared packages estáveis | ✅ `packages/` (config, core, design-tokens, shared-data, storage) |
| Mobile scaffold e MVP em uso | ✅ Android validado |
| Push native e beta interno funcionando | ✅ |
| Build web atual verde | ⚠️ Verificar antes de Sprint 7.3 |
| Testes web verdes | ⚠️ Verificar antes de Sprint 7.3 |
| **Maintainer aprovou** | ❓ **PENDENTE** |
| `apps/mobile/app.config.js` e `eas.json` estáveis | ✅ |

## Decisão Proposta

Executar os Sprints 7.1–7.5 conforme `plans/backlog-native_app/EXEC_SPEC_HIBRIDO_FASE7_MIGRACAO_WEB_APPS_WEB.md`:

- **Sprint 7.1** — Preparação e freeze de escopo
- **Sprint 7.2** — Criação de `apps/web` e package local
- **Sprint 7.3** — Move físico e reparo de tooling
- **Sprint 7.4** — Compatibilidade root e automações
- **Sprint 7.5** — Deploy e validação final

## Riscos Principais

1. **Vercel rootDir** — build da web muda de `./` para `apps/web`; requer ajuste no dashboard Vercel e `vercel.json`
2. **Coexistência api/ + apps/web** — Vercel precisa resolver serverless em `api/` e frontend em `apps/web` simultaneamente
3. **Aliases Vite** — todos os aliases `@/*` devem ser reconfigurados em `apps/web/vite.config.js`
4. **Vitest** — `vitest.config.js` raiz precisa atualizar `include` para apontar para `apps/web/src/**`
5. **Scripts root** — `npm run dev`, `npm run build` precisam de wrappers retrocompatíveis

## Como Aprovar

O maintainer deve explicitamente confirmar: **"Aprovado — executar Fase 7"**

Após aprovação, alterar `status: proposed` → `status: accepted` neste arquivo e iniciar Sprint 7.1.
