# ADR-003: Groq Prompt Caching Optimization Strategy

**Status:** Accepted | **Date:** 2026-03-20 | **Source:** PR #412
**Tags:** chatbot, groq, optimization, caching

## Context
Chatbot was making redundant Groq API calls across web and Telegram interfaces, increasing costs.

## Decision
Implement unified prompt caching layer to deduplicate Groq API requests and reduce costs.

## Implementation
- Create `cachedGroqClient` wrapper around Groq SDK
- Cache prompt+parameters → response mapping
- Invalidate cache on model/temperature changes
- Share cache between web and Telegram clients

## Consequences
✅ Reduced API costs
✅ Faster response times from cache hits
⚠️ Memory usage increases with cache size
⚠️ Stale cached responses if prompt semantics change

## Testing
- Measure API call count before/after
- Verify cache hit rates
- Validate response consistency
