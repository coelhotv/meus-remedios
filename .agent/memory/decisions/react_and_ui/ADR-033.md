## Decisão

Frontend DEVE implementar validação de cache pós-carregamento para evitar exibição de protocolos expirados que ainda residem em AsyncStorage/SWR após transições automáticas de data (meia-noite) ou retorno do modo offline.

## Rationale

- **Problema:** Em aplicativos mobile/PWA, quando a data local muda (meia-noite) ou o usuário volta online, o cache pode estar desatualizado
- **Contexto:** Protocolos com `end_date` já passado devem desaparecer automaticamente, mesmo que persistidos no AsyncStorage
- **Impacto:** Sem validação pós-load, usuário vê protocolos "expirados" que deveriam ter desaparecido

## Solução

1. Implementar `validateCacheTimestamp()` ou similar em hooks de sincronização (ex: `useSyncState`)
2. Ao reconnect ou date transition: validar cada entry contra `today` local
3. Remover entries com `end_date < today` do AsyncStorage antes de renderizar
4. Emit state update para UI refletir mudança

## Implementação

- **Hook:** `useSyncState` + event listeners para 'online'/'midnight'
- **Services:** `adherenceService`, `protocolService` com validação integrada
- **Testing:** Timezone-aware mocks em `vitest` para simular date transitions
