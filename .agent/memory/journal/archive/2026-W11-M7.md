# Sprint M7 — N+1 Query Elimination (2026-03-13)

## Deliverables

✅ **M7.1:** `calculateAllProtocolsAdherence()` refactored from N+1 to batch query pattern
- Before: `Promise.all(protocols.map(...))` = 11 queries with 10 protocols
- After: 1 batch query `SELECT protocol_id` + client-side `Map` grouping (O(M) once, not O(M)×N)
- Commit: `8e52ca9` (squashed)

✅ **M7.2:** `calculateAdherence()` uses HEAD request for count-only queries
- Before: `select('*')` transferred all columns (~500 bytes/row)
- After: `select('*', { count: 'exact', head: true })` = zero data bytes transferred
- Saves ~270 lines × 10 protocols = ~1.35MB per call

- **MEDIUM:** Extract `_getDateRangeForPeriod()` helper to eliminate duplication across `calculateAdherence()` and `calculateAllProtocolsAdherence()`

## Learnings & Anti-Patterns

### Documented in `.memory/anti-patterns.md`

- **AP-P09:** N+1 Query Pattern in services → use batch query + Map grouping (R-118)
- **AP-P10:** `select('*')` when only need count → use HEAD request with `count: 'exact'` (R-119)
- **AP-P11:** `useCallback` deps with state in ref callback → use `deps: []` + `useRef` for flags (R-120)

### Key Insights

1. **Safari Performance Trace (2026-03-13):** Revealed 15 simultaneous queries blocking main thread 100ms+ with 10 active protocols. Root cause: N+1 pattern + unnecessary data transfer.

2. **Date Range Extraction:** Both `calculateAdherence()` and `calculateAllProtocolsAdherence()` had identical date calculation logic. Extraction to `_getDateRangeForPeriod()` prevents future bugs from timezone-dependent date handling (common source of ±1 day off-by-one errors).

3. **HEAD Requests:** `select('*', { count: 'exact', head: true })` is Supabase idiom for count-only queries. Zero network overhead compared to fetching full rows when only count matters.

4. **Gemini Review Quality:** Both HIGH suggestions were critical:
   - `endDate` parameter consistency prevents midnight boundary bugs in timezone-sensitive environments (GMT-3 offset in Brazil)
   - Duplication elimination reduces future maintenance burden
