## Learnings & Anti-Patterns

### R-W20: Branch Creation is Workflow Step 1.0, Not Optional
**Rule:** Feature branch MUST be created BEFORE any code change.
**Why:** Protects against accidental main commits, ensures audit trail, enables PR review process.
**Applied:** AP-A01 created with mandatory sequence.

### AP-A01: Skipping Branch Creation
**Anti-Pattern:** Modifying code on main branch or creating branch after changes
**Consequence:** Code on main without review, lost audit trail, violates R-060/R-065
**Prevention:** Add Step 1.0 as FIRST action in deliver-sprint workflow
