# CON-005: logService.createLog(data)

**Status:** stable
**Type:** service-method
**File:** src/shared/services/api/logService.js

## Signature

```javascript
async createLog(data: LogInput): Promise<Log>
```

## Input Type (LogInput)

```typescript
{
  medicine_id: string (uuid)
  quantity_taken: number (always in pills, max 100)
  taken_at: ISO 8601 timestamp (MUST use parseLocalDate)
  notes?: string
  treatment_plan_id?: string (uuid)
  protocol_id?: string (uuid)
}
```

## Returns

```typescript
{
  id: string (uuid)
  medicine_id: string
  user_id: string
  quantity_taken: number
  taken_at: ISO 8601 timestamp
  created_at: ISO 8601 timestamp
  notes: string | null
}
```

## Consumers

- `src/features/adherence/hooks/useRegisterDose.js` — dose registration
- `src/features/stock/services/stockService.js` — stock decrease on dose log
- `src/views/Dashboard.jsx` — quick log from dashboard
- `server/bot/callbacks/conversational.js` — Telegram bot dose logging

## Breaking Change Definition

- Input parameter type change (added required fields)
- Return type change
- Database schema change requiring migration
- Timestamp handling change (timezone issues)

## Migration Guide Template

```javascript
// Must ensure all consumers pass parseLocalDate() for taken_at
import { parseLocalDate } from '@/utils/dateUtils'

const registerDose = async (medicineId, quantity, dateString) => {
  // BEFORE: new Date(dateString) — wrong for BR timezone
  // AFTER: parseLocalDate(dateString) — correct
  const log = await logService.createLog({
    medicine_id: medicineId,
    quantity_taken: quantity,
    taken_at: parseLocalDate(dateString),
    notes: 'Logged from dashboard'
  })
  return log
}
```

## Related ADRs

- ADR-NNN: Dosage in pills, not mg (max 100)
- R-020: Never use new Date('YYYY-MM-DD')
