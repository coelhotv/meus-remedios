# CON-018: calculateAdherenceStats(logs, protocols, days, offsetDays)

## Signature
`calculateAdherenceStats(logs: Array, protocols: Array, days: number = 30, offsetDays: number = 0) -> Object`

## Parameters
- `logs`: Array of mediation logs (DB objects with `taken_at`).
- `protocols`: Array of active protocols.
- `days`: Window of analysis (e.g., 7 days).
- `offsetDays`: Starting point for the window (D0 = today, D7 = 1 week ago). Default is 0.

## Returns
```json
{
  "score": number, // 0-100 adherence percentage
  "expected": number, // Total doses expected in period
  "taken": number, // Total doses taken correctly (within tolerance)
  "takenAnytime": number, // Doses taken regardless of window
  "currentStreak": number // Current consecutive successful days
}
```

## Why
This is the core business logic shared between Web and Mobile to calculate floating adherence windows (Trends). The addition of `offsetDays` allows for comparative analysis (Week current vs Week previous) without duplicate logic.

## Usage
```javascript
// Current week
const stats = calculateAdherenceStats(logs, protocols, 7, 0);
// Previous week
const statsPrev = calculateAdherenceStats(logs, protocols, 7, 7);
```
