## Signature

```typescript
export async function dispatchNotification({
  userId: string,
  kind: 'dose_reminder' | 'stock_alert' | 'daily_digest',
  payload: { title: string, body: string, deeplink: string, metadata: object },
  channels: string[],                          // ['telegram'] | ['mobile_push'] | ['telegram', 'mobile_push']
  context: { correlationId?: string },
  repositories: { preferences, devices },
  bot: TelegramBotInstance,
  expoClient: ExpoClientInstance,
}) → Promise<{
  success: boolean,
  channels: ChannelResult[],
  totalDelivered: number,
  totalFailed: number,
}>
```

## ChannelResult Shape

```typescript
{
  channel: 'telegram' | 'mobile_push',
  success: boolean,
  attempted: number,
  delivered: number,
  failed: number,
  deactivatedTokens: string[],
  errors: { message: string, code?: string }[],
}
```

## Invariants

1. **Isolation:** Falha de um canal não cancela outros canais (via Promise.allSettled)
2. **Validation:** Input validado via Zod schema antes de qualquer operação (dispatchInputSchema)
3. **Logging:** Todos os logs incluem `correlationId` obrigatoriamente (R-087)
4. **Deactivation:** Tokens com erro permanente (DetectedError) são imediatamente desativados via `repositories.devices.deactivateByToken()`
5. **Retry:** Erros transitórios (timeout, 5xx) são retornados em `errors[]` sem automático retry (DLQ é responsibility de caller em Sprint 6.4)

## Breaking Changes

Qualquer mudança na assinatura (adicionar required param, remover param, alterar tipo de retorno) é **breaking change** e requer:
- Nova ADR documentando a mudança
- Atualização de todos os callers (jobs em Sprint 6.4)
- Feature flag de rollback ativada antes de deploy

## Non-Breaking Changes

- Adicionar optional field a `ChannelResult`
- Adicionar novo erro a `errors[]` com novo código
- Adicionar novo `kind` suportado (com validação estrita)

## Linked Rules

- R-087: correlationId em todos os logs
- R-042: service_role para deactivateByToken (server-side only)
- R-041: Nunca process.exit(); throw Error

## Examples

### Telegram Only (preference='telegram')

```js
const result = await dispatchNotification({
  userId: 'user-123',
  kind: 'dose_reminder',
  payload: { title: 'Hora do remédio', body: 'Tome Losartana', deeplink: 'dosiq://today', metadata: {} },
  channels: ['telegram'],
  context: { correlationId: 'corr-001' },
  repositories: { ... },
  bot: telegramBotInstance,
  expoClient: expoInstance,
})
// → { success: true, channels: [{channel: 'telegram', success: true, attempted: 1, delivered: 1, ...}], totalDelivered: 1, totalFailed: 0 }
```

### Both (preference='both')

```js
const result = await dispatchNotification({
  ...
  channels: ['telegram', 'mobile_push'],
  ...
})
// → { success: true, channels: [{channel: 'telegram', ...}, {channel: 'mobile_push', ...}], totalDelivered: 2, totalFailed: 0 }
```

### Push Fails, Telegram Succeeds (falha isolada)

```js
const result = await dispatchNotification({
  channels: ['telegram', 'mobile_push'],
  ...
})
// → { success: false, channels: [{channel: 'telegram', success: true, delivered: 1, ...}, {channel: 'mobile_push', success: false, failed: 2, ...}], totalDelivered: 1, totalFailed: 2 }
```
