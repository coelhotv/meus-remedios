## Contract

**Function:** `resolveChannelsForUser({ userId, repositories })`

**Input:**
- `userId` (string, uuid): ID do usuário Supabase
- `repositories` (object): Dependency injection com:
  - `repositories.preferences.getByUserId(userId)` → preferência
  - `repositories.preferences.hasTelegramChat(userId)` → boolean
  - `repositories.devices.listActiveByUser(userId, provider)` → array de devices

**Output:**
- Array of strings: `[]`, `['telegram']`, `['mobile_push']`, ou `['telegram', 'mobile_push']`

## Decision Logic

| Preferência | Telegram? | Devices? | Resultado |
|-------------|-----------|----------|-----------|
| none | T/F | T/F | `[]` |
| telegram | T | T/F | `['telegram']` |
| telegram | F | T/F | `[]` |
| mobile_push | T/F | T | `['mobile_push']` |
| mobile_push | T/F | F | `[]` |
| both | T | T | `['telegram', 'mobile_push']` |
| both | T | F | `['telegram']` |
| both | F | T | `['mobile_push']` |
| both | F | F | `[]` |

## Usage (Sprint 6.2+)

```js
// Em dispatcher ou job (após Sprint 6.2)
const channels = await resolveChannelsForUser({ userId, repositories })
if (channels.length === 0) {
  return { success: true, delivered: 0, failed: 0 } // nenhum canal ativo
}
const result = await dispatchNotification({ userId, channels, ... })
```
