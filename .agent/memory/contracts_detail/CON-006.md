# CON-006: useCachedQuery(key, fetcher, options)

**Status:** stable
**Type:** react-hook
**File:** src/shared/hooks/useCachedQuery.js

## Signature

```javascript
function useCachedQuery<T>(
  key: string,
  fetcher: () => Promise<T>,
  options?: CacheOptions
): {
  data: T | undefined
  error: Error | undefined
  isLoading: boolean
  mutate: () => Promise<void>
  isValidating: boolean
}
```

## Options

```typescript
{
  revalidateOnFocus?: boolean (default: true)
  revalidateOnReconnect?: boolean (default: true)
  dedupingInterval?: number (default: 2000ms)
  focusThrottleInterval?: number (default: 5000ms)
  errorRetryCount?: number (default: 3)
  errorRetryInterval?: number (default: 5000ms)
  staleTime?: number (default: 60000ms)
}
```

## Consumers

- `src/features/dashboard/hooks/useDashboardContext.jsx` — dashboard data

- `src/features/stock/hooks/useStockData.js` — stock data

## Breaking Change Definition

- Hook signature change (parameter order, type)
- Return object shape change
- Option names/defaults change
- Cache invalidation strategy change

## Usage Pattern

```javascript
const { data: medicines, error, isLoading, mutate } = useCachedQuery(
  'medicines',
  () => medicineService.getAll(),
  { staleTime: 60000 }
)

if (isLoading) return <ViewSkeleton />
if (error) return <ErrorBoundary error={error} />
return <MedicinesList medicines={data} onRefresh={mutate} />
```

## Performance Considerations

- Deduplication: multiple calls with same key reuse one fetch
- Manual mutations: `mutate()` invalidates cache instantly
- Stale-while-revalidate: returns cached data while revalidating
- Never disable deduplication without ADR
