Two separate crashes prevent standard URL polyfills on Hermes: (1) react-native-url-polyfill wrapper accesses NativeModules.BlobModule.getConstants() which is undefined in Expo Go; (2) whatwg-url-without-unicode uses Symbol.for('[webidl2js] constructor registry') which is never initialized by Hermes. Both produce 'Cannot read property get of undefined' but from different sources.
