fact: A partir do Expo SDK 53, o plugin oficial do Firebase v21 falha ao injetar código no AppDelegate.swift devido à ausência da string âncora 'self.moduleName'. Além disso, o CocoaPods exige modular headers para dependências do Firebase ao usar Swift. O uso de um Config Plugin local (withFirebaseFix.js) é necessário para garantir a inicialização do Firebase e a configuração do Podfile.
valid_for:
  - mobile
  - ios
  - firebase
source: Session 2026-04-18 debugging
---

No Expo SDK 53, o arquivo `AppDelegate.swift` gerado automaticamente remove a atribuição `self.moduleName`, o que quebra a Regex de detecção do plugin `@react-native-firebase/app` v21. Isso resulta no Firebase não sendo inicializado, apesar do build completar (com avisos).

A solução robusta envolve:
1. Implementar um **Config Plugin local** (`withFirebaseFix.js`) que usa `withAppDelegate` para injetar `FirebaseApp.configure()` e o import `FirebaseCore` manualmente no `AppDelegate.swift`.
2. Usar `withDangerousMod` no mesmo plugin para garantir que `use_modular_headers!` seja adicionado ao `Podfile`, resolvendo erros de 'Swift pod FirebaseCoreInternal depends upon GoogleUtilities'.
3. Registrar este plugin no `app.config.js`.
