# R-042 — Service Role Authentication [HIGH]

## Rule
Any endpoint using the `service_role` Supabase key (which bypasses RLS) MUST have authentication. Verify admin identity via Supabase Auth + `ADMIN_CHAT_ID`.

## Why
PR #73 security fix. A `service_role` endpoint was exposed without authentication, allowing any caller to bypass row-level security.

## Verification pattern
```javascript
// Always verify before using service_role
const { data: { user } } = await supabase.auth.getUser(token)
if (!user || user.id !== ADMIN_USER_ID) {
  return res.status(403).json({ error: 'Forbidden' })
}
// Only now use service_role operations
```
