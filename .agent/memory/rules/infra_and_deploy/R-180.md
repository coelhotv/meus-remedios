# [R-180] Monorepo Local EAS Build Inclusion

## Context
When running `eas build --local` in a monorepo, the EAS CLI uses `git archive` to package the context. If your root `.gitignore` excludes configuration files (like Firebase `.plist` or `.json`), they will be missing from the local build folder, causing `ENOENT` errors.

## Rule
In monorepos using local EAS builds, a `.easignore` file MUST exist at the **repository root** (and optionally in the app directory). 
- You MUST use the `!` (negation) pattern to explicitly force the inclusion of ignored config files.
- Example: `!apps/mobile/GoogleService-Info-production.plist`

## Rationale
Ensures that sensitive but necessary build-time credentials (which are correctly git-ignored) are still passed to the local native compilation process.
