# R-041 — No process.exit() in Serverless [HIGH]

## Rule
Never use `process.exit()` in Vercel serverless functions.

**Correct approach:** throw `new Error()` to signal failure.

**Conditional dotenv:** `if (process.env.NODE_ENV !== 'production') { require('dotenv').config() }`

## Why
`process.exit()` crashes the entire serverless container, not just the request. In production this caused the bot to go completely offline.

## Pattern
```javascript
// Correct
if (!token) throw new Error('TELEGRAM_TOKEN not set')

// Wrong
if (!token) process.exit(1)
```

## Consolidated anti-patterns
- `AP-010` — call `process.exit()` inside serverless code

Use this rule as the canonical guidance for failure signaling in serverless handlers. The AP remains as a historical negative example.
