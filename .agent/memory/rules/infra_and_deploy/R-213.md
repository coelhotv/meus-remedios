# R-213 — Wildcard exports em packages de workspace compartilhado

Ao criar re-exports de sub-arquivos de um workspace package (`@dosiq/core/schemas/X.js`), o `package.json` do package DEVE declarar wildcard exports:

```json
"exports": {
  "./schemas/*": "./src/schemas/*.js",
  "./utils/*": "./src/utils/*.js"
}
```

**Why:** Vite 7 + Vitest respeitam o `exports` field do package.json. Sem o wildcard, imports como `@dosiq/core/schemas/logSchema.js` falham com "Missing specifier" mesmo que o Vite alias aponte para o diretório correto.

**How to apply:** Em stubs de re-export dentro do monorepo, preferir paths relativos diretos (`'../../../../packages/core/src/schemas/X.js'`) em vez de imports de package (`@dosiq/core/schemas/X.js`). Paths relativos bypassam alias e package.json exports — funcionam em qualquer ambiente (local e CI). O wildcard no exports field resolve localmente mas falha em CI com versões específicas de Vite/Node.
