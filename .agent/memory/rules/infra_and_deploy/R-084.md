# R-084 — Private Storage Authentication [HIGH]

## Rule
When accessing private storage (Vercel Blob, S3, etc.), always include authentication headers. Never assume public access.

## Pattern
```javascript
// CORRECT
const response = await fetch(blobUrl, {
  headers: { 'Authorization': `Bearer ${process.env.BLOB_READ_WRITE_TOKEN}` }
})

// WRONG — assumes public access
const response = await fetch(blobUrl)
```

## Source
Sprint 7 (PR #150) — 403 Forbidden on blob downloads

## Consolidated anti-patterns
- `AP-S04` — fetch private storage without auth headers

Use this rule as the canonical guidance for private object storage access. The AP remains as a historical shorthand.
