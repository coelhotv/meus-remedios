# R-132: Logging em api/*.js para Visibilidade Vercel

**Summary:** Estruturar logs no entry point api/*.js, NÃO em server/bot/**. Node.js server context é invisível para Vercel logging (VMs diferentes).

**Applies to:** vercel

**Tags:** infra, process

## Rationale

Logs written in the wrong runtime context are invisible where operators expect to inspect them.

## Implementation Notes

Place observability-critical logs in Vercel entry points when the execution path goes through `api/*.js`.
