# R-040 — Vercel API Rewrites [HIGH]

## Rule
Always add explicit rewrites in `vercel.json` for new API routes:
- The catch-all `/(.*) → /index.html` MUST be the LAST rewrite
- Use `:id` syntax for dynamic parameters (not `[id]`)

## Why
DLQ 405 errors from new API routes missing explicit rewrites. The catch-all intercepts API requests if it appears before the API rule.

## Example vercel.json
```json
{
  "rewrites": [
    { "source": "/api/medications/:id", "destination": "/api/medications/[id]" },
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

## Consolidated anti-patterns
- `AP-011` — add API route without explicit Vercel rewrite

Use this rule as the canonical guidance for rewrite safety in `vercel.json`. The AP remains as a shorthand incident label.
