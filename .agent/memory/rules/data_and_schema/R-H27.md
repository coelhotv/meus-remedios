# R-H27: Native Adaptive Complexity (Persona Logic)

## Context
O Dosiq utiliza um sistema de densidade de interface adaptativa para atender diferentes perfis de pacientes (Dona Maria vs Carlos).

## Thresholds (Sincronizados com Web)
- **Simples (Dona Maria)**: ≤ 3 medicamentos ativos.
- **Moderado/Complexo (Carlos)**: > 3 medicamentos ativos.

## Implementation (Mobile)
A lógica deve ser calculada dinamicamente no componente de visualização (ex: `TodayScreen.jsx`) baseada nos dados do snapshot (`medicines` record).

### Regras de Exibição
1. **Modo Simples**:
   - Ocultar turnos (shifts) que não possuem doses pendentes ou concluídas para o dia.
   - Focar no "Card Deck" UI (essencial e limpo).
2. **Modo Complexo**:
   - Mostrar todos os turnos principais (`Manhã`, `Tarde`, `Noite`).
   - Se um turno estiver vazio, exibir o header em estado mutado (`isDisabled`) e uma mensagem interna de "Vazio".
   - Funciona como um "Control Panel" denso.

## Por que?
Pacientes com poucos medicamentos sentem a interface "vazia" ou "quebrada" se muitos slots de tempo ficarem desocupados. Pacientes complexos precisam de uma estrutura fixa para orientação temporal e comparação de densidade.
