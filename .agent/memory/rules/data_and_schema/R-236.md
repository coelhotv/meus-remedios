# R-236 — Mutation documenta TODOS os caches que invalida (matrix explícita)

**Regra**: toda mutation (`useXMutation.create/update/delete/customAction`) deve listar inline (comment ou JSDoc) TODOS os snapshots AsyncStorage que invalida. Esquecer um cache adjacente = bug latente.

**Why**: ver [[AP-168]]. `toggleActive` invalidou só `protocols-snapshot`, esqueceu `treatments-snapshot` — listagem servia cache stale. Detectado pelo Gemini PR #570; trivial mas custou 1 ciclo de review.

**How to apply**:

```js
// useXMutation.js — pattern obrigatório

/**
 * toggleActive — alterna flag `active` do tratamento.
 * Caches invalidados:
 *   - @dosiq/protocols-snapshot (detail/useProtocol/useProtocols)
 *   - @dosiq/treatments-snapshot (listagem/useTreatments — usa cache key próprio
 *     pra incluir grouping + counts)
 * Caches NÃO invalidados (intencionalmente):
 *   - @dosiq/medicines-snapshot (toggle não afeta lista de medicamentos)
 */
const toggleActive = useCallback(async (id, nextValue) => {
  const result = await protocolService.update(id, { active: nextValue })
  await Promise.all([
    AsyncStorage.removeItem(PROTOCOLS_CACHE_KEY),
    AsyncStorage.removeItem(TREATMENTS_CACHE_KEY),
  ]).catch(() => {})
  return result
}, [/* deps */])
```

**Quando NÃO aplicar**: mutations puramente locais (ex: toggle de UI state que não toca server) não tem cache pra invalidar.

**Pattern preventivo**: ao implementar mutation nova, fazer grep por outras hooks que consultam mesma tabela DB e listar TODOS os cache keys deles.

```bash
rtk grep -rn "CACHE_KEY\|snapshot" apps/mobile/src/features/<dominio>/hooks/
```

**Sweep retroativo recomendado** (Fase 3 ou fix-pack):
- `useProtocolMutation.create/update` esquecem treatments-snapshot
- `useProtocolDelete` esquece treatments-snapshot
- `useMedicineMutation.*` — auditar idem
