# R-192: Supabase Array Filtering (.contains)

## Contexto
Ao buscar dados no Supabase (PostgreSQL) que possuem colunas do tipo array (ex: `time_schedule` em `protocols`), a filtragem deve ocorrer sempre que possível no lado do banco de dados para evitar sobrecarga de memória no servidor e tráfego de rede desnecessário.

## Regra
1. **Filtre no Banco de Dados:** Utilize o operador `.contains('<coluna>', [<valor>])` para colunas de array.
2. **Evite Filtros em Memória:** Nunca carregue todos os registros ativos (`.eq('active', true)`) para filtrar por horário no JavaScript, a menos que a complexidade da lógica impeça a tradução para SQL.

## Exemplo (Correto)
```javascript
const { data } = await supabase
  .from('protocols')
  .select('id, name')
  .eq('user_id', userId)
  .contains('time_schedule', [hhmm]);
```

## Exemplo (Incorreto)
```javascript
const { data: all } = await supabase
  .from('protocols')
  .select('id, name, time_schedule')
  .eq('user_id', userId);

const filtered = all.filter(p => p.time_schedule.includes(hhmm));
```

## Benefícios
- Redução drástica do payload de resposta.
- Melhoria de performance (TTFB).
- Alinhamento com as diretrizes de performance mobile do projeto.
