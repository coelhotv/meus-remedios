# R-020 — Timezone — Local Dates [CRITICAL]

## Rule
**ALWAYS** use `parseLocalDate(dateStr)` from `@utils/dateUtils` or `new Date(dateStr + 'T00:00:00')`.

**NEVER** use `new Date('YYYY-MM-DD')` — this creates UTC midnight, which is 21:00 of the PREVIOUS day in GMT-3.

## Why
Production adherence score bug: `new Date('2026-02-18')` created 2026-02-17T21:00:00 in GMT-3. Adherence scores were attributed to the wrong day.

## Examples
```javascript
// Correct
import { parseLocalDate } from '@utils/dateUtils'
const date = parseLocalDate('2026-02-18')

// Also correct
const date = new Date('2026-02-18' + 'T00:00:00')

// WRONG — midnight UTC = 21:00 Feb 17 in GMT-3
const date = new Date('2026-02-18')
```

## Applies to
Any date string in YYYY-MM-DD format from Supabase, API responses, or user input.
