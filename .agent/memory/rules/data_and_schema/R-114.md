# R-114: Zero Hours on Date Boundaries for Timezone Consistency

**Summary:** When setting date boundaries for filtering, call .setHours(0,0,0,0) to prevent off-by-one errors when comparing local dates with ISO-string UTC timestamps.

**Applies to:** all

**Tags:** timezone, data

## Rationale

Date-boundary comparisons drift when residual hours/minutes leak into local-vs-UTC checks.

## Implementation Notes

Normalize day-boundary `Date` objects with `.setHours(0, 0, 0, 0)` before filtering or comparison.

## Consolidated anti-patterns
- `AP-A04` — compare local dates with residual hours/minutes

Use this rule as the canonical guidance for normalized date boundaries. The AP remains as a historical anti-example.
