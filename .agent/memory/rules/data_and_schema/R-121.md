# R-121: Zod Validation for Service Parameters

**Summary:** ALL service functions accepting input params MUST validate with Zod safeParse() + schema. Never use manual typeof/Array.isArray() checks.

**Applies to:** zod

**Tags:** zod, data, safety

## Rationale

Manual parameter checks are inconsistent and incomplete. Zod gives one validation path with clearer guarantees and error handling.

## Implementation Notes

Validate service inputs at the boundary with Zod schemas and `safeParse()`.
