# R-129: String Comparison para Datas em Hot Loops

**Summary:** Em loops com >100 iterações sobre datas YYYY-MM-DD, usar comparação de strings (dateStr < startStr) ao invés de new Date(). Strings YYYY-MM-DD são lexicograficamente ordenáveis.

**Applies to:** all

**Tags:** performance, timezone

## Rationale

Constructing large volumes of `Date` objects in hot loops is avoidable when lexicographic comparison is sufficient.

## Implementation Notes

Use string comparison only for already-normalized `YYYY-MM-DD` values and only where date arithmetic is not needed.
