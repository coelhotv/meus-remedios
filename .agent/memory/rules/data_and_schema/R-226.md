# R-226 — Filtro FIFO: `entry_type != 'legacy_unrecoverable'`

## Rule

Em qualquer RPC ou query de consumo FIFO de estoque, o filtro de `entry_type` DEVE ser:

```sql
AND entry_type != 'legacy_unrecoverable'
```

**NUNCA:**
```sql
AND entry_type = 'purchase'
```

## Why

Entradas `adjustment` com `quantity > 0` representam pílulas físicas reais:
- Backfill de "Dose excluída / Ajuste de dose" (pré-migração 20260402)
- Criadas por `apply_manual_stock_adjustment` (ajuste manual positivo)

O único tipo que deve ser excluído do FIFO é `legacy_unrecoverable` — entradas sem histórico recuperável, sempre com `quantity <= 0`.

Filtrar por `entry_type = 'purchase'` cria "phantom stock": `medicine_stock_summary` exibe saldo > 0 mas o FIFO lança "Estoque insuficiente". Ver AP-142.

## How to Apply

Em toda query de consumo/disponibilidade de estoque:

```sql
-- Pre-check de disponibilidade
SELECT COALESCE(SUM(quantity), 0)
FROM public.stock
WHERE medicine_id = p_medicine_id
  AND user_id = v_user_id
  AND quantity > 0
  AND entry_type != 'legacy_unrecoverable';  -- ← correto

-- Loop FIFO
SELECT * FROM public.stock
WHERE medicine_id = p_medicine_id
  AND user_id = v_user_id
  AND quantity > 0
  AND entry_type != 'legacy_unrecoverable'   -- ← correto
ORDER BY purchase_date ASC, created_at ASC, id ASC
FOR UPDATE;
```

## Invariant

`medicine_stock_summary` e `consume_stock_fifo` devem estar em sincronia:
> Se a view inclui uma entrada no total, o FIFO deve poder consumi-la.
