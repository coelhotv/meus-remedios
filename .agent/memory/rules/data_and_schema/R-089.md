# R-089 — Database Column Existence Check [HIGH]

## Rule
When writing INSERT statements, verify all columns exist in the target table. Never assume based on previous versions or documentation.

## Prevention
- Keep migration files synchronized with code
- Use TypeScript types generated from DB schema
- Add integration tests that actually insert data

## Source
Sprint 7 (PR #146) — `review_data` column didn't exist
