# [R-185] Consistência de Estados de Carregamento Nativo

## Descrição
Evita a percepção de instabilidade ou montagem parcial da interface ao carregar dados pela primeira vez em uma tela.

## Gatilho
Criação ou refatoração de Views/Screens mobile.

## Regra
- **LoadingState**: Telas devem usar o componente `LoadingState` centralizado enquanto os dados obrigatórios não estiverem disponíveis.
- **Animações de Mount**: Evitar animações de entrada (como `framer-motion` slide-up) em telas que carregam dados assincronamente, para evitar que a animação ocorra antes ou durante o spinner de carregamento.
- O spinner deve ser a única transição visual de "entrada" aprovada para o fluxo principal de dados.
