# R-180: Native Header Standardization (Santuário Pattern)

## Context
Following the Phase 8 mobile redesign ("Santuário Terapêutico"), all main view headers must maintain a specific visual weight and spacing to ensure platform consistency and a premium feel.

## Rule
Headers in the 4 primary tabs (`Hoje`, `Tratamentos`, `Estoque`, `Perfil`) and secondary configuration screens MUST follow the established "Santuário" tokens.

### Main Tab Headers:
- **Title**: `fontSize: 28`, `fontWeight: '800'`, `letterSpacing: -0.5`.
- **Subtitle**: `fontSize: 16`, `color: '#74777f'`.
- **Containers**: `paddingHorizontal: 20`, `paddingVertical: 16`, `marginBottom: 8`.

### Secondary/Config Headers (Telegram Model):
- **Back Button**: Must use the "← Voltar" text pattern (not just an icon).
- **Separator**: Must include a `borderBottomWidth: 1` with `colors.border.light` to clearly define the navigation boundary.
- **Internal Padding**: Consistent with main screens.

## Why
Avoids visual fragmentation between features (e.g., Stock used to be smaller than Dashboard) and professionalizes the navigation experience.

## Examples

### ✅ Correct (Main Tab)
```jsx
const styles = StyleSheet.create({
  header: {
    paddingHorizontal: 20,
    paddingVertical: 16,
    marginBottom: 8,
  },
  title: {
    fontSize: 28,
    fontWeight: '800',
    letterSpacing: -0.5,
  }
})
```

### ✅ Correct (Config/Telegram Pattern)
```jsx
<View style={styles.header}>
  <TouchableOpacity onPress={goBack} style={styles.backButton}>
    <Text style={styles.backButtonText}>← Voltar</Text>
  </TouchableOpacity>
  <Text style={styles.headerTitle}>Details</Text>
</View>
```
