# R-157: Weekly Audit: Web/Bot Code Drift

**Summary:** Auditar semanalmente divergências entre lógica do Telegram bot (server/bot/) e web app (src/features/). Bot deve usar exatamente os mesmos RPCs e services. Drift silencioso causa bugs em produção (ex: FIFO stock restoration).

**Applies to:** telegram, supabase

**Tags:** telegram, process, safety

## Rationale

The bot and the web app can drift even when they implement the same domain behavior. Regular audits reduce silent production mismatch.

## Implementation Notes

Review shared business flows weekly and verify both surfaces still rely on the same canonical logic.
