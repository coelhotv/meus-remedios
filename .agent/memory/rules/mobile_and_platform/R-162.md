# R-162: Hermes URL patch — usar patch inline em vez de pacotes externos

**Criado em:** 2026-04-12
**Contexto:** React Native com Supabase em Hermes/Expo Go SDK 53

---

## A Regra

Para projectos React Native que usam Supabase (ou qualquer lib que use `URL` getters/setters),
usar o patch inline de `polyfills.js` em vez de qualquer pacote `react-native-url-polyfill`.

---

## Por Que Esta Regra Existe

O Hermes tem implementação parcial de `URL` — `href` funciona mas getters como
`.protocol`, `.hostname`, etc. lançam "not implemented". O Supabase Realtime também
faz setters (`url.protocol = 'wss:'`).

`react-native-url-polyfill` (v2, v3) substitui a implementação global de `URL` mas usa
módulos nativos não disponíveis em Expo Go, causando crash imediato.

Testado e confirmado que não existe versão funcional de react-native-url-polyfill
para Expo Go SDK 53.

---

## Como Aplicar

### 1. Criar `apps/mobile/polyfills.js`

```js
// polyfills.js — deve ser importado ANTES de qualquer outro módulo
global.SharedArrayBuffer = global.SharedArrayBuffer || global.ArrayBuffer

;(function patchHermesURL() {
  if (typeof URL === 'undefined') return
  const proto = URL.prototype

  function needsPatch(prop) {
    try {
      new URL('https://example.com')[prop]
      return false
    } catch (e) {
      return typeof e.message === 'string' && e.message.includes('not implemented')
    }
  }

  function parseHref(href) {
    const m = href.match(
      /^([a-z][a-z0-9+\-.]*:)\/\/([^/:?#@]*)(?::(\d+))?(\/[^?#]*)?(\?[^#]*)?(#.*)?$/i
    )
    if (!m) {
      return { protocol: '', hostname: '', port: '', pathname: '/', search: '', hash: '', origin: 'null' }
    }
    const protocol = (m[1] || '').toLowerCase()
    const hostname = m[2] || ''
    const port = m[3] || ''
    return {
      protocol, hostname, port,
      pathname: m[4] || '/',
      search: m[5] || '',
      hash: m[6] || '',
      origin: `${protocol}//${hostname}${port ? ':' + port : ''}`,
    }
  }

  function define(prop, getter, setter) {
    if (!needsPatch(prop)) return
    const descriptor = { get: getter, configurable: true }
    if (setter) descriptor.set = setter
    Object.defineProperty(proto, prop, descriptor)
  }

  define('protocol',
    function () { return parseHref(this.href).protocol },
    function (val) {
      const p = String(val).endsWith(':') ? String(val) : String(val) + ':'
      this.href = this.href.replace(/^[a-z][a-z0-9+\-.]*:/i, p)
    }
  )
  define('hostname',
    function () { return parseHref(this.href).hostname },
    function (val) {
      const { protocol, port, pathname, search, hash } = parseHref(this.href)
      this.href = `${protocol}//${val}${port ? ':' + port : ''}${pathname}${search}${hash}`
    }
  )
  // ... (ver polyfills.js para implementação completa)
})()
```

### 2. Importar como PRIMEIRO import em `index.js`

```js
import './polyfills'  // PRIMEIRO — antes de qualquer lib
import { AppRegistry } from 'react-native'
import App from './App'
AppRegistry.registerComponent('main', () => App)
```

### 3. NÃO instalar react-native-url-polyfill

```json
// package.json — NÃO adicionar:
// "react-native-url-polyfill": "^2.0.0"  // CRASH
// "react-native-url-polyfill": "^3.0.0"  // CRASH
```

## Propriedades a Patchear

| Propriedade | Getter | Setter | Notas |
|-------------|--------|--------|-------|
| `protocol` | ✅ | ✅ | Setter obrigatório — Supabase Realtime faz `url.protocol = 'wss:'` |
| `hostname` | ✅ | ✅ | |
| `port` | ✅ | ✅ | |
| `host` | ✅ | ✅ | hostname:port |
| `pathname` | ✅ | ✅ | |
| `search` | ✅ | ✅ | |
| `hash` | ✅ | ✅ | |
| `origin` | ✅ | ❌ | Read-only por spec |
| `username` | ✅ | ❌ | Retorna '' |
| `password` | ✅ | ❌ | Retorna '' |
| `searchParams` | ✅ | ❌ | Cria URLSearchParams do search |
