# R-174 — Monorepo Mobile Dependency Resolution (Metro Proxy Aliases)

## Context
In a monorepo (npm workspaces), dependencies like `react` and `react-native` may exist in multiple `node_modules` (root and workspace). Metro might bundle multiple instances, causing the fatal error `TypeError: Cannot read property 'useState' of null`.

## Rule
All mobile apps in the monorepo MUST configure `metro.config.js` with a **Proxy Alias** to force critical dependencies to resolve to the local `node_modules` of the app workspace.

## Implementation (metro.config.js)

```javascript
const path = require('path')
const projectRoot = __dirname

// ... setup config ...

config.resolver.extraNodeModules = new Proxy(
  {},
  {
    get: (target, name) => {
      // Força estas bibliotecas a serem carregadas APENAS da pasta local
      if (['react', 'react-native', '@react-native-async-storage/async-storage'].includes(name)) {
        return path.resolve(projectRoot, 'node_modules', name)
      }
      return target[name]
    },
  }
)
```

## Why Proxy?
Unlike a static object, a `Proxy` allows Metro to handle requested modules dynamically while intercepting critical ones. This is more robust than simple path aliases when dealing with complex dependency trees in workspaces.
