# R-117: Lazy Views + manualChunks for Code Splitting

**Summary:** All non-critical views MUST use React.lazy() + import(). Combine with vite.config.js manualChunks for vendor/feature isolation.

**Applies to:** react

**Tags:** react, performance

## Rationale

Loading all views eagerly increases bundle size and hurts first-load performance. Code splitting keeps the critical path smaller.

## Implementation Notes

Use `React.lazy()` for non-critical views and keep Vite chunking aligned with feature boundaries.
