# [R-215] Native Dependency Sync for Cross-Platform Components

## Context
When reusing shared components from `@shared` in the `apps/mobile` workspace, especially components that were originally designed for web or shared usage, they might rely on libraries that have native dependencies (e.g., `expo-linear-gradient`, `expo-blur`, `react-native-svg`).

## Problem
Even if the component is imported from a shared location, the native app's `package.json` must explicitly list the native dependency to ensure:
1. Metro bundler can find and link the native code.
2. CI linting tools (which scan `package.json` for used imports) do not fail with "missing dependency" errors.
3. EAS builds correctly include the required native modules.

## Rule
**When reusing web/shared components in mobile that use native modules like `expo-linear-gradient`, the dependency MUST be explicitly added to `apps/mobile/package.json` to prevent runtime crashes and CI lint errors.**

## Examples

### ❌ Incorrect
Adding a component like `PriorityActionCard` (which uses `LinearGradient`) to a mobile screen without checking `apps/mobile/package.json`.

### ✅ Correct
```bash
cd apps/mobile
npm install expo-linear-gradient
```
And verifying the import in the mobile screen refers to the library correctly.

## Verification
1. Run `npm run lint` in `apps/mobile`.
2. Ensure the app starts in Expo Go or development build without "Module not found" errors.
