# R-128: Promise Coalescence para Auth Roundtrips

**Summary:** Funções que fazem HTTP roundtrip para auth (getUser(), getSession()) DEVEM implementar coalescência: se já há chamada em voo, reutilizar a mesma promise.

**Applies to:** supabase

**Tags:** performance, supabase

## Rationale

Repeated auth calls during the same load sequence create avoidable latency and duplication.

## Implementation Notes

Cache the in-flight promise and reuse it until resolution.
