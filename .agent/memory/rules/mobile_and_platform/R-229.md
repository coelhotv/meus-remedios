# [R-229] `deepEqual` em hooks de formulário DEVE tratar `Date` por valor

## Definição
Qualquer comparação de igualdade profunda usada para detectar dirty state em formulários (ex: `useFormState` mobile) DEVE comparar objetos `Date` via `getTime()`. Comparação genérica por `Object.keys` falha silenciosamente — `Object.keys(new Date())` retorna `[]`, então datas distintas são consideradas iguais e `isDirty` retorna `false`.

## Implementação
```javascript
const deepEqual = (a, b) => {
  if (a === b) return true
  // Date precisa de comparação por valor — Object.keys(Date) retorna []
  if (a instanceof Date && b instanceof Date) return a.getTime() === b.getTime()
  if (a == null || b == null) return a === b
  if (typeof a !== 'object' || typeof b !== 'object') return a === b
  const ka = Object.keys(a)
  const kb = Object.keys(b)
  if (ka.length !== kb.length) return false
  return ka.every((k) => deepEqual(a[k], b[k]))
}
```

## Por que importa
Forms com `FormDatePicker`/`FormTimePicker` (Fase 0 P.1) armazenam valores como `Date`. Sem o branch acima:
- `isDirty` reportava `false` mesmo após o usuário alterar a data.
- Botão "Salvar" podia ficar incorretamente desabilitado em UIs que dependem de `isDirty`.
- Mutations puladas silenciosamente.
