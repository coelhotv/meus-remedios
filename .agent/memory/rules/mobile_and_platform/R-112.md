# R-112: Adherence Must Sum quantity_taken, Not Count Logs

**Summary:** Calculate adherence as sum(quantity_taken)/expected*100, NOT count(logs)/expected*100. A patient taking 2 pills/dose shouldn't have 50% when taking 1 log/day.

**Applies to:** all

**Tags:** data

## Rationale

One log entry does not always equal one expected unit. The numerator must reflect actual quantity taken.

## Implementation Notes

Use `quantity_taken` aggregation for adherence math whenever dose quantity matters.

## Consolidated anti-patterns
- `AP-A02` — count logs instead of summing `quantity_taken`

Use this rule as the canonical guidance for adherence numerator semantics. The AP remains as a fast negative reference.
