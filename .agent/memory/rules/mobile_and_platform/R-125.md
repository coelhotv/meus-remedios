# R-125: Cache SWR para Queries de Adesão

**Summary:** Queries do adherenceService DEVEM usar cachedAdherenceService. Keys: adherence:summary:{period}, adherence:daily:{days}, adherence:pattern.

**Applies to:** all

**Tags:** performance, supabase

## Rationale

Adherence data is queried often and benefits from SWR-style caching. Reusing the cache layer avoids duplicate network work.

## Implementation Notes

Route adherence queries through the cached service and keep cache-key conventions consistent.
