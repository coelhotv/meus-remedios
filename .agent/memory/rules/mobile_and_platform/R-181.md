# [R-181] iOS App Tracking Transparency (ATT) Compliance

## Context
Apple rejects apps that declare data collection for tracking in App Store Connect privacy labels but do not implement the native permission prompt.

## Rule
iOS apps that collect User ID, Device ID, or Advertising Data for tracking purposes (including future monetization/ads) MUST:
1.  Implement the `expo-tracking-transparency` framework.
2.  Add `NSUserTrackingUsageDescription` in `ios.infoPlist` inside `app.config.js`.
3.  Call `requestTrackingPermissionsAsync()` as early as possible (e.g., in `AppRoot.jsx`).

## Rationale
Prevents rejection under **Guideline 5.1.2(i)** and prepares the app for legal monetization and analytics tracking.
