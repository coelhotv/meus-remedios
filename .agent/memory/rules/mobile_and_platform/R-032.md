# R-032 — shouldSendNotification() Includes Logging [MEDIUM]

## Rule
`shouldSendNotification()` already calls `logNotification()` internally. NEVER call `logNotification()` explicitly after it returns `true` — causes duplicate notification logs.

## Why
Duplicate logs caused notification history to show double entries, confusing users and breaking analytics.

## Pattern
```javascript
// Correct
if (shouldSendNotification(user, medicine)) {
  await sendTelegramMessage(...)
  // logNotification already called internally ↑
}

// Wrong — double log
if (shouldSendNotification(user, medicine)) {
  await sendTelegramMessage(...)
  await logNotification(...)  // ← duplicate!
}
```
