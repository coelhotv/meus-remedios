# R-175: Timezone-Resilient Snapshots

Snapshots de dados para uso offline que requerem segregação por dia (ex: tela Hoje) **DEVEM** incluir um campo `localDay` explicitamente capturado via `getTodayLocal()`.

## Por que?
O campo `capturedAt` padrão é gerado em formato ISO (UTC). Se um snapshot é capturado às 22:00 (GMT-3), o ISO será `YYYY-MM-DD+1T01:00:00Z`. Ao comparar o `split('T')[0]` deste ISO com o `getTodayLocal()` atual, haverá um descasamento, causando a limpeza incorreta de logs ou exibição de dados do dia errado.

## Como Aplicar
No momento do `AsyncStorage.setItem`:

```javascript
const snapshot = {
  data: newData,
  capturedAt: new Date().toISOString(),
  localDay: getTodayLocal() // ✅ OBRIGATÓRIO
}
```

Na recuperação (`getItem`):
```javascript
const today = getTodayLocal();
const isDaySegregated = parsed.localDay !== today; // ✅ Comparação Robusta
```
