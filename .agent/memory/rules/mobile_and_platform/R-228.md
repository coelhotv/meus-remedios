# [R-228] Padrão Serializável de Retorno entre Telas (React Navigation)

## Definição
Em fluxos onde uma tela B precisa retornar dados para uma tela A no React Navigation (RN mobile), passar **somente valores serializáveis** em `route.params`. Callbacks (`onSelect`, `onConfirm`, etc.) **NÃO** podem ir em `route.params`. Usar o padrão `returnRoute` + `merge:true` + `setTimeout(0)` defer no consumidor.

## Por que
- React Navigation emite warning `Non-serializable values were found in the navigation state` quando callbacks são passados em params — eles não sobrevivem a deep links nem serialização de estado.
- Closures em callbacks tornam-se stale quando a tela A é re-renderizada após push de B.
- Lint `react-hooks/set-state-in-effect` exige defer quando o consumer aplica params recebidos em `useEffect`.

## Padrão correto
```jsx
// Tela A — abre B passando rota de retorno
navigation.navigate('AnvisaSearchScreen', { returnRoute: 'MedicineFormScreen' })

// Tela B — devolve dado para A via navigate + merge
function handleSelect(item) {
  navigation.navigate({
    name: route.params.returnRoute,
    params: { selectedMedicine: item },
    merge: true,
  })
}

// Tela A — consome dado com setTimeout(0) defer
useEffect(() => {
  if (!route.params?.selectedMedicine) return
  const t = setTimeout(() => {
    form.setValues({ ...form.values, ...route.params.selectedMedicine })
    navigation.setParams({ selectedMedicine: undefined })
  }, 0)
  return () => clearTimeout(t)
}, [route.params?.selectedMedicine])
```
