# R-051 — Validate Before Push [HIGH]

## Rule
- During development: `npm run validate:quick` (lint + changed tests only)
- Before creating PR: `npm run validate` (lint + all tests)

## Why
PRs with failing tests waste review cycles and block CI. Catching failures locally before push is faster and less disruptive.

R-074 (validate:agent for agent sessions — agents must NOT use validate:quick)
