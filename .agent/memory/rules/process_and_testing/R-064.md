# R-064: Protocolos de Interação Agente-Humano

**Summary:** Define o fluxo de aprovação obrigatório para perguntas (propor) e pedidos (executar mas mostrar antes de commit). Proíbe merge/push sem autorização.

## Rationale

Garante que a proatividade do agente não resulte em alterações não planejadas ou pushes prematuros para produção, mantendo o controle final com o usuário.

## Implementation Notes

1.  **Perguntas**: Limitar-se à análise e proposição.
2.  **Pedidos**: Executar a lógica, mas validar e mostrar o resultado antes de persistir em commit.
3.  **Merge/Push**: Bloqueado até autorização humana expressa.
