# [R-212] Otimização de Comandos CLI via RTK

## Contexto
O projeto utiliza o utilitário `rtk` (Rust Token Killer) como um proxy para comandos CLI comuns (`git`, `npm`, `vitest`, etc.). Este utilitário filtra a saída dos comandos para remover ruídos repetitivos e metadados desnecessários, economizando entre 60% e 90% do contexto de tokens em sessões de IA.

## Regra
Sempre que for necessário executar comandos no terminal, utilize o prefixo `rtk`.

## Exemplos
- **Testes:** `rtk vitest`, `rtk npm test`, `rtk test <cmd>`
- **Lint & Build:** `rtk lint` (agrupa por regra/arquivo), `rtk tsc`, `rtk next build`, `rtk prettier --check .`
- **Git (Modo Compacto):** `rtk git status`, `rtk git log -n 10`, `rtk git add`, `rtk git commit -m "msg"`, `rtk git push`
- **JSON & Deps:** `rtk json config.json` (estrutura sem valores), `rtk deps` (resumo de dependências)
- **Logs:** `rtk log app.log` (logs deduplicados), `rtk summary <comando>` (resumo heurístico)
- **Geral:** `rtk <comando>`, `rtk err <comando>` (apenas erros)

## Exceções
Use `rtk proxy <comando>` apenas se precisar ver a saída bruta completa para depuração de problemas no próprio `rtk`.

## Benefícios
- Redução drástica de custo de tokens.
- Foco em falhas e mensagens de erro relevantes.
- Terminal mais limpo e produtivo.
