# R-071 — Mock External Services in Hook Tests [CRITICAL]

## Rule
Any hook test rendering a Provider that calls external services (Supabase, API) MUST mock those services at the module level with `vi.mock()`.

Without mocks, the hook fires real network calls that hang indefinitely in test environments.

## Why
useDashboardContext.test.jsx had no mocks for medicineService, protocolService, logService. Tests hung indefinitely waiting for network responses that never resolved.

## Pattern
```javascript
// Required at top of test file that uses DashboardProvider
vi.mock('@medications/services/medicineService', () => ({
  medicineService: { getAll: vi.fn().mockResolvedValue([]) },
}))
vi.mock('@protocols/services/protocolService', () => ({
  protocolService: { getAll: vi.fn().mockResolvedValue([]) },
}))
```
