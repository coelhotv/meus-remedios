# R-077 — Cancel setInterval in Tests [HIGH]

## Rule
Global `setInterval` (e.g., garbage collection) MUST be cancellable and restarted in test hooks.

Export `cancelGarbageCollection()` / `restartGarbageCollection()` from modules that use global intervals.

## Why
2026-02-23. GC intervals running during tests caused memory accumulation. Without explicit cancellation, workers accumulated state across test files.

## Pattern
```javascript
// Module: queryCache.js
let gcInterval = setInterval(garbageCollect, 60000)
export function cancelGarbageCollection() {
  if (gcInterval) { clearInterval(gcInterval); gcInterval = null }
}
export function restartGarbageCollection() {
  if (!gcInterval) { gcInterval = setInterval(garbageCollect, 60000) }
}

// Test setup
beforeAll(() => { cancelGarbageCollection() })
afterAll(() => { restartGarbageCollection() })
```

## Consolidated anti-patterns
- `AP-T04` — leave global `setInterval` running across tests

Use this rule as the canonical guidance for interval lifecycle in test environments. The AP remains as a quick negative lookup.
