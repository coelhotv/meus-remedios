# R-178: Test Coverage Integrity (Zero Silent Debt)

Ao criar novas pastas de código ou features fora de `src/` (ex: `server/`, `api/`, `scripts/`), o agente **DEVE** garantir que a configuração das ferramentas de teste e lint (Vitest, Jest, ESLint) seja atualizada para incluir esse novo diretório.

## Por que isso é importante?
- **Falsa Segurança:** Testes que existem no sistema de arquivos mas não rodam no CI criam uma ilusão de cobertura.
- **Regressões Silenciosas:** Mudanças podem quebrar lógicas "testadas" sem que o desenvolvedor perceba, pois o comando de teste global passa com sucesso (ignorando os arquivos fora do scan).

## Alerta de Anti-Pattern (Silent Test Debt)
NUNCA considere uma feature "testada" se o novo arquivo `.test.js` não for capturado pelo comando oficial `npm run test`.
- Se você criou um teste em uma nova pasta e o contador de testes não subiu, você está em débito técnico silencioso.

## Como Aplicar
1. Verifique `vitest.config.js` ou `jest.config.js`.
2. Adicione o glob pattern correspondente ao array `include` (ex: `'server/**/*.test.js'`).
3. Execute `npm run test` e confirme a execução dos novos casos.
