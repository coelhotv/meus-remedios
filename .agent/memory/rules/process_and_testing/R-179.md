# R-179 — Proteção de Dados de Produção [CRITICAL]

## Regra
É terminantemente proibido alterar dados reais de usuários humanos durante sessões de teste, automação de browser ou depuração manual, especialmente em contas identificadas como de produção (ex: domínios pessoais @live.com, @gmail.com do proprietário).

**Protocolo de Segurança:**
1. Se o ambiente estiver conectado ao Supabase de Produção, use apenas leitura (`SELECT`).
2. Para testes de escrita (`INSERT/UPDATE/DELETE`), utilize exclusivamente contas com prefixo `test-` ou domínio `@example.com`.
3. Se for necessário manipular dados para validar uma UI, desfaça a alteração IMEDIATAMENTE após o teste.

## Por Que
Incidente em 2026-04-20: O agente alterou a data de expiração de um medicamento real ("Omega 3") para validar filtros de dashboard, impactando o controle de saúde do usuário real. Violação grave de R-062 (Qualidade) e ética de manipulação de dados.

## DEVFLOW Note
Esta regra é de cumprimento obrigatório para todos os agentes. O não cumprimento pode levar ao bloqueio imediato da sessão por risco de integridade.
