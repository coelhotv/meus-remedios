# R-074: Use validate:agent for Agent Sessions

**Summary:** Agents MUST use npm run validate:agent (10-min kill switch). Never use validate:quick or validate:full from agent sessions.

**Applies to:** all

**Tags:** testing, process

## Rationale

Agent sessions need a predictable validation command with timeout protection. Using alternative validation commands increases the chance of long-running sessions, inconsistent checks, or skipped critical safeguards.

## Implementation Notes

Use `npm run validate:agent` as the default validation command for agent-driven changes.
