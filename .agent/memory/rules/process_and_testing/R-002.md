# R-002 — Path Alias Verification [HIGH]

## Rule
When you see `@adherence/services/x`, verify it resolves to `src/features/adherence/services/x` (NOT `src/services/api/`). Always check `vite.config.js` for alias definitions before assuming a path.

## Why
Production bugs caused by modifying the wrong file because alias resolution was assumed, not verified. `@feature/x` aliases can be misleading.

## How to Verify
```bash
grep -A 20 "alias" vite.config.js
```
