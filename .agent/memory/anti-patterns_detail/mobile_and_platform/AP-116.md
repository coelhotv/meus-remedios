# AP-116 — Double Notification Cold Start Processing

## Problem

Em `usePushNotifications()`, o hook processa `getLastNotificationResponseAsync()` para suportar deep linking em cold start. Se o useEffect tiver `[session]` ou `[user]` na deps array, toda vez que o usuário faz login/logout, a notificação é reprocessada.

Resultado: **navegação duplicada, visual stutter, possível tela errada em exibição**.

## Evidence

- Sprint N1.4 (PR #499): Identified by Gemini Code Assist review
- Trigger: logout + login (credenciais expiram, usuário refaz auth)
- Symptom: Same deep link é navegado 2x → screen state conflita, animação cheia de ruído

## How It Happens

```js
// ❌ ERRADO
const [session, setSession] = useState()

useEffect(() => {
  const response = await getLastNotificationResponseAsync()
  if (response) navigate(...)
}, [session]) // BUG: reexecuta em cada mudança de session
```

Ao fazer logout + login:
1. `session` muda (null → new session)
2. useEffect reexecuta
3. `getLastNotificationResponseAsync()` retorna mesma resposta anterior
4. navigate() é chamada 2x na mesma notification → duplicate state

## Prevention

Usar `useRef` flag para guardar que cold start foi processado uma vez:

```js
// ✅ CORRETO
const coldStartProcessed = useRef(false)

useEffect(() => {
  if (coldStartProcessed.current) return
  
  const response = await getLastNotificationResponseAsync()
  if (response) {
    navigate(...)
    coldStartProcessed.current = true
  }
}, []) // Empty deps: roda uma vez na montagem
```
