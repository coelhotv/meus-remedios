# AP-095 — Skip local quality gates before push

**Category:** Quality Gates  
**Status:** active  
**Related Rule:** R-051  
**Applies To:** all  
**Trigger Count:** 1 (2026-04-08T21:45:00Z)

## Problem

Skipping local `npm run lint && npm run build` before pushing to branch results in:
- Vercel preview build fails (broken imports, build errors)
- PR cannot be merged until fixed
- Wasted review cycles (reviewer sees failing checks)
- TDZ/ReferenceError bugs slip through because static analysis doesn't catch runtime behavior

```bash
# ❌ WRONG — no local validation
git push origin feature/wave-17/fix
gh pr create ...
# Later: Vercel preview build fails ❌

# ✅ CORRECT — validate locally first
npm run lint      # Catch linting errors
npm run build     # Catch import errors, build config errors
git push origin feature/wave-17/fix
gh pr create ...
# Vercel preview passes ✓
```

## Impact

- **Cost:** 20-30 minutes of wasted PR review cycles
- **Discovery latency:** Errors found in Vercel preview instead of locally
- **Reviewer friction:** Failing checks block PR before code review even starts
- **Runtime bugs:** Static analysis (ESLint, DEVFLOW reviewing) cannot detect:
  - TDZ ReferenceErrors from hook order violations
  - Service response validation failures
  - Build-time import resolution issues

## Prevention

**Mandatory Pre-Push Checklist:**

```bash
# Always run BEFORE git push
npm run lint        # ~5 sec
npm run build       # ~3 sec
npm run validate:agent  # ~10 sec (critical tests only)
```

Only push after all three pass.

**Why This Matters**

- Lint catches formatting + obvious code issues
- Build catches import resolution + module errors  
- validate:agent catches runtime logic bugs in critical services/schemas/hooks
- **Total time: 18 seconds** vs. 30 minutes of wasted review

## Testing

Always run `npm run build` to verify:
- All imports resolve (no `ERR_MODULE_NOT_FOUND`)
- vite.config.js manualChunks references valid files
- CSS compilation works
- Dynamic imports don't have typos
