# AP-114: Validação de Entrada Silenciosa (Missing Zod)

## Sintoma
Implementar lógica de negócio em Services (Server-side) ou Features sem envolver uma camada de validação de esquema (Zod), confiando que o chamador enviará dados limpos.

## Por que é um Anti-Padrão?
1. **Violação da Regra de Ouro #10:** Todo input em Services deve ser validado.
2. **Corrupção de Dados:** Dados inválidos podem chegar ao banco de dados ou causar falhas silenciosas durante decremento de estoque.
3. **Dificuldade de Depuração:** Erros de tipo ou formato são propagados para camadas profundas (como RPCs) sem contexto claro do erro original.

## Como Corrigir
1. **Use Schemas Canônicos:** Importe os schemas do pacote `@dosiq/core/src/schemas`.
2. **Fail Fast:** Utilize `.safeParse()` ou funções de validação no início do Service.

## Exemplo Real (Sprint 1.2)
O `medicineLogService.createMany` foi inicialmente implementado sem validar o array `logs`, violando a integridade do sistema de estoque.
