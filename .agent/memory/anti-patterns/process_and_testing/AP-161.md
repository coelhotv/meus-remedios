# [AP-161] DEVFLOW state.json counter drift entre distillations

## Sintoma
Sessions individuais adicionam R-NNN/AP-NNN/ADR-NNN/CON-NNN aos respectivos `[CLASS]_INDEX.md` markdown E criam detail files em `memory/[class]/[category]/`, mas o counter correspondente em `.agent/state.json` (`memory.rules_count`, `memory.anti_patterns_count`, etc.) NÃO é incrementado consistentemente.

Resultado: `state.json` fica dessincronizado dos índices ao longo de várias sessões. Distill subsequente, se ler `state.json` como source of truth para narrative, produz inconsistências matemáticas (ex: "+9 rules adicionadas mas counter subiu apenas +1") — detectadas posteriormente pelo reviewer LLM no PR de distill.

## Histórico no projeto
- **2026-05-16 (PR #559)**: Distill pós-Fase 1 reportou "9 rules + 10 APs + 2 ADRs adicionadas" mas counters em state.json subiram apenas +1/+1/+1 (e contracts caíram 21→20). Gemini detectou as inconsistências aritméticas. Reconciliação retroativa confirmou que state.json estava stale desde `last_distillation = 2026-04-26` — 3 semanas de drift escondido.

## Causa-raiz
Em DEVFLOW, regras/APs/ADRs são criados em **C5** (pós-coding) com bump em `state.json.memory.journal_entries_since_distillation`, mas os counters `rules_count` / `anti_patterns_count` / `decisions_count` / `contracts_count` em `state.json` NÃO são atualizados automaticamente por C5 — só por D6 (final do distill). Como distill só roda quando threshold é atingido, drift acumula entre distills.

Adicionalmente, D5 ("Autonomous Self-Cleaning") foi escrito como OPCIONAL no SKILL — agente pode skippar para "save time", deixando o problema escondido por mais 1 ciclo.

## Correção (aplicada em 2026-05-16 — Spike Pre-Fase-2)

1. **D5 vira MANDATORY DEEP SCAN** (SKILL.md atualizado em `~/SKILLS/devflow/SKILL.md`):
   - Reconciliar state.json counters contra `grep -cE '^- \*\*\[<PREFIX>-' [CLASS]_INDEX.md`
   - Emitir bloco `reconciliation: { before, after, delta, interpretation }` no journal entry de distillation
   - Auditar contract drops específicamente (delta negativo em contracts é sinal de remoção; documentar)

2. **C5 não precisa mudar** (manter atualização de counter incremental fora do escopo — overhead maior que benefício; D5 reconcilia em batch).

## Como evitar manualmente (até D5 estar formalizado em todos workflows)
Em qualquer commit que modifica `[CLASS]_INDEX.md`, NÃO mexer em counters de state.json. Distill subsequente reconcilia. Documentar bumps no commit message para audit trail.
