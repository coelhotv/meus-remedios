# AP-169 — Branch local outdated antes de spawn → sub-agent duplica arquivos

**Sintoma**: Criei branch `feat/crud-protocols-t2-3b` a partir de `feat/crud-protocols` local DESATUALIZADO (não fiz pull após merge do PR-A). Spawn Sonnet pra refatorar web service detectou que branch divergia de main e "portou manualmente" os arquivos da factory (que já existiam em origin) pro novo branch — gerando duplicação ao tentar push.

**Root cause**: ritual `git fetch + verificar local == origin` não foi feito antes de criar branch nova. Sonnet rodando em working dir igual ao meu sentiu a divergência e tentou corrigir copiando arquivos — comportamento defensivo mas que escondeu a causa real (branch base outdated).

**Por que perigoso**: gasta tempo do agent + risco de conflitos não-óbvios + commits órfãos se push for forçado. Em pior caso, duas implementações da mesma factory entram em divergência sem detecção.

**Fix imediato**: reset hard do branch local pra origin, recrir branch t2-3b, stash apenas as mudanças web-only, reaplicar.

**Regra preventiva**: antes de criar branch nova OU spawn cavecrew que toque arquivos compartilhados:
1. `rtk git fetch origin`
2. Verificar branch base: `git log --oneline origin/<base>..HEAD` deve ser vazio (local == origin)
3. SE não vazio, `git pull` ou reset hard antes de criar branch

Brief de spawn DEVE mencionar branch base esperado pra agent detectar drift.

**Anti-correlato**: NÃO instruir cavecrew a fazer "git pull" autônomo — agent pode pular branches ou conflitar com working dir local.
