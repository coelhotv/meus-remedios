# AP-170 — Alias adicionado em `vitest.config.js` não propaga para configs secundários

**Sintoma**: `validate:agent` (CI gate kill switch) falhava silenciosamente com `TypeError: createProtocolRepository is not a function` mesmo após adicionar alias `@dosiq/core` ao `vitest.config.js`. Causa: existe um SEGUNDO config (`vitest.critical.config.js`) que CI usa especificamente, e ele NÃO foi atualizado.

**Root cause**: configs vitest do projeto não compartilham `baseConfig` — cada um declara aliases inline. Mudança em um não propaga.

**Por que perigoso**: tests passam localmente (rodam `vitest.config.js`), CI falha (roda `vitest.critical.config.js`). Diff invisível causa surpresa em PR review.

**Fix aplicado**: replicar alias em ambos `vitest.config.js` + `vitest.critical.config.js`.

**Regra preventiva**: sempre que adicionar/mudar alias em um vitest config, grep no repo por outros configs vitest e propagar:

```bash
rtk find apps/ -name 'vitest*.config.js' -type f
```

Refator futuro (não bloqueante): extrair `baseConfig` shared entre os configs vitest e fazer cada um spread + override.

- `apps/web/vitest.config.js` (dev)
- `apps/web/vitest.critical.config.js` (CI kill switch)
