# AP-094 — Reference handler in useEffect before declaration

**Category:** React Hooks  
**Status:** active  
**Related Rule:** R-010  
**Applies To:** react, hooks  
**Trigger Count:** 1 (2026-04-08)

## Problem

`ReferenceError: Cannot access 'loadCardData' before initialization` — useEffect tries to call a handler (useCallback) that hasn't been declared yet, causing TDZ (Temporal Dead Zone) crash.

```jsx
// ❌ WRONG — useEffect calls loadCardData before it exists
const [data, setData] = useState()

useEffect(() => {
  loadCardData()  // ReferenceError! loadCardData not yet declared
}, [loadCardData])

const loadCardData = useCallback(async () => {
  // ...
}, [])
```

## Impact

- Runtime crash: white screen, no console error clarity initially
- View doesn't render at all
- Browser shows "Uncaught ReferenceError" in console
- User cannot access feature

## Prevention

**Correct Hook Order:** States → Handlers (useCallback) → Effects (useEffect)

Handlers MUST be declared BEFORE any effect that references them:

```jsx
// ✅ CORRECT — handlers before effects
const [data, setData] = useState()

const loadCardData = useCallback(async () => {
  // ...
}, [])

const handleSave = useCallback(async (updatedData) => {
  // ...
}, [])

useEffect(() => {
  loadCardData()  // Now safe — handler exists
}, [loadCardData])
```

## Why This Happens

R-010 says: States → useMemo → useEffect → Handlers

But when an effect **calls** a handler, the standard order breaks. The effect needs the handler to exist. Solution: Move handlers before effects when they're used.

## Testing

Always test React components locally before push:
```bash
npm run lint
npm run build
```

Build will catch import errors, but runtime validation needs manual component test in browser.
