# [AP-130] SyntaxError: The requested module does not provide an export named 'X' (ESM)

## Contexto
Ocorreu durante a refatoração da Wave 2 no serviço de notificações. O arquivo `dispatchNotification.js` importava `kindSchema` de `buildNotificationPayload.js`, mas este último não exportava o símbolo explicitamente como um named export, embora o utilizasse internamente.

## Sintoma
O deploy no Vercel falha em runtime com:
`SyntaxError: The requested module '../payloads/buildNotificationPayload.js' does not provide an export named 'kindSchema'`

## Causa Raiz
Em ESM (Node.js 20+), re-exports ou exportações implícitas não são resolvidas se não houver um `export const/type` explícito no módulo de origem. Durante a modularização, o `kindSchema` foi movido para um helper privado mas o consumidor continuou tentando importá-lo do arquivo principal que não o re-expunha.

## Como Evitar
1. **Contrato de Exportação:** Sempre verifique se símbolos importados por outros módulos estão na lista de `export` do arquivo index ou do arquivo de origem.
2. **Validação de Runtime:** Testes unitários de integração que importam o módulo real (não mockado) pegam esse erro antes do deploy.
3. **Padrão de Re-export:** Se um símbolo é definido em um helper `_schema.js` e deve ser público, use `export { symbol } from './_schema.js'` no arquivo principal.
