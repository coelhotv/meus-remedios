## Problema

Usar `var(--color-primary, #006a5e)` ou qualquer `var(--token)` em arquivos de `packages/core` ou outro pacote compartilhado entre web e mobile causa erro em runtime no React Native:

```
"var(--color-primary, #006a5e)" is not a valid color or brush
```

O React Native não tem motor CSS — não resolve custom properties, nem usa o fallback da sintaxe `var(--token, fallback)`.

## Exemplo quebrado

```js
// ❌ packages/core/src/utils/notificationIconMapper.js
color: 'var(--color-primary, #006a5e)',   // web OK, RN crash
color: 'var(--color-warning, #d97706)',
```

## Fix

```js
// ✅ Usar o valor hex diretamente
color: '#006a5e',
color: '#d97706',
```

## Regra de ouro para pacotes compartilhados

Qualquer arquivo em `packages/` que é consumido pelo mobile (React Native) **não pode** conter:
- `var(--*)` — CSS custom properties
- `calc(...)` — expressões CSS
- Qualquer outra sintaxe CSS que dependa de motor de browser

## Alternativas para tematização cross-platform

- Importar tokens de `packages/design-tokens` como valores JS (ex: `import { colors } from '@dosiq/design-tokens'`)
- Passar a cor como prop do componente e aplicar `var(--token)` apenas na camada web
- Manter dois mappers separados se as plataformas tiverem sistemas de cor divergentes

## Impacto real

Sprint 8.3 — `notificationIconMapper.js` (usado por `NotificationCard.jsx` web e `NotificationItem.jsx` mobile) tinha 6 entradas com `var(--color-*)`. Resultado: dezenas de warnings no console do Expo, um por card renderizado.
