# [AP-156] `schema.pick({[field]: true})` em validação isolada descarta refinements de objeto

## Sintoma
Hook genérico de validação isolada por campo (`validateField` em `useFormState`) usa `schema.pick({ [field]: true }).safeParse({ [field]: value })` para validar apenas o campo modificado. Quando o schema tem `.refine()` ou `.superRefine()` no **nível do objeto** (validação cross-field), o `pick` descarta esses refinements silenciosamente — validações cross-field nunca disparam em `onBlur`.

## Exemplo
```javascript
const medicineSchema = z.object({
  start_date: z.date(),
  end_date: z.date(),
}).refine(
  (data) => data.end_date >= data.start_date,
  { message: 'Data fim deve ser maior ou igual a início', path: ['end_date'] }
)

// Errado — refine descartado
const picked = medicineSchema.pick({ end_date: true })
picked.safeParse({ end_date: dateAnterior }) // success (refine ignorado)
```

## Correção
Parsear o objeto completo + filtrar issues por campo:
```javascript
const validateField = (schema, field, value, allValues) => {
  const result = schema.safeParse({ ...allValues, [field]: value })
  if (result.success) return null
  const issue = result.error.issues.find((i) => i.path[0] === field)
  return issue ? issue.message : null
}
```

## Impacto
- Forms com regras cross-field deixavam de validar em blur (falso positivo).
- Detectado por Gemini em revisão do PR #552 (Fase 0 final mãe→main).

## Aplicação
- `apps/mobile/src/shared/hooks/useFormState.js` — commit 38f43cf1
- Repensar qualquer factory de validação que use `pick` para isolar campos.
