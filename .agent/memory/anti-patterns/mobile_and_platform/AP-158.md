# [AP-158] Callbacks em `route.params` quebram React Navigation + criam stale closures

## Sintoma
Passar funções (callbacks) em `route.params` ao navegar entre telas no React Navigation. Tela origem passa `onSelect: (item) => {...}` esperando que a tela destino chame; tela destino chama o callback ao confirmar a ação.

```jsx
// Anti-pattern
navigation.navigate('AnvisaSearchScreen', {
  onSelect: (medicine) => form.setValues(medicine), // ❌
})
```

## Por que é problema
- React Navigation emite warning `Non-serializable values were found in the navigation state` (válido — params devem ser serializáveis para suportar deep links, persistência de estado, deep linking).
- O callback captura o snapshot da **tela origem no momento do navigate** — se a tela é re-renderizada depois (state mudou, prop mudou), o callback chama `form.setValues` com referência velha → bugs de stale closure.
- Não funciona se o app for restaurado a partir de deep link / URL — params serializáveis sobrevivem, callbacks não.

## Correção (R-228)
Usar padrão serializável `returnRoute` + `navigate({ name, params, merge: true })` + `setTimeout(0)` defer no consumidor. Ver [R-228] para template completo.

## Impacto
- Warn em dev mode (pollui console).
- Bug latente sob deep linking.
- Stale closures sob re-render da tela origem.

## Aplicação
- Detectado e corrigido em `AnvisaSearchScreen` ↔ `MedicineFormScreen` durante Fase 0 P.2 (PR #549).
- Reforçado em P.3 (PR #551) com `setTimeout(0)` defer para satisfazer lint `react-hooks/set-state-in-effect` no consumer.
