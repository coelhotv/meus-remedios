# [AP-162] Import de `@expo/vector-icons` carrega ~4 MB de fonts cumulativos no bundle

## Sintoma
Importar QUALQUER família do meta-pacote `@expo/vector-icons` (ex: `import { Ionicons } from '@expo/vector-icons'`) registra TODOS os ~20 fonts do pacote como assets do app (MaterialCommunityIcons, FontAwesome6_*, Ionicons, Fontisto, MaterialIcons, etc.), mesmo que apenas 1 família seja usada.

Resultado: ~4 MB de fonts adicionados ao bundle/dist para uso de 5-10 ícones.

## Evidência no Dosiq (2026-05-16, Spike Pre-Fase-2)
`expo export --platform ios/android`:
- Total `dist/assets/`: **4.6 MB**
- Decomposição: vector-icons fonts ≈ 4 MB; Comfortaa 555 kB; icon app 281 kB
- 5 telas auth/landing importam `Ionicons` de `@expo/vector-icons`:
  - `apps/mobile/src/screens/LoginScreen.jsx`
  - `apps/mobile/src/screens/SignupScreen.jsx`
  - `apps/mobile/src/screens/ForgotPasswordScreen.jsx`
  - `apps/mobile/src/screens/ResetPasswordScreen.jsx`
  - `apps/mobile/src/screens/LandingScreen.jsx`

Resto do app usa `lucide-react-native` (SVG-based, zero fonts assets).

## Correção (3 opções)

### Opção A — Migrar para `lucide-react-native` (recomendada)
Consistência com o resto do app. Esforço ~30min. Eliminação completa de `@expo/vector-icons`. Economia ~4 MB.

```diff
- import { Ionicons } from '@expo/vector-icons'
+ import { Eye, EyeOff, Mail, Lock, ArrowLeft } from 'lucide-react-native'

- <Ionicons name="eye-outline" size={20} color="#666" />
+ <Eye size={20} color="#666" />
```

Após migração de todas as 5 telas:
```bash
npm uninstall --workspace @dosiq/mobile @expo/vector-icons
```

### Opção B — Import específico por família
Mantém Ionicons (única família usada), descarrega resto.

```diff
- import { Ionicons } from '@expo/vector-icons'
+ import Ionicons from '@expo/vector-icons/Ionicons'
```

Economia ~3.5 MB (mantém apenas Ionicons.ttf 443 kB).

### Opção C — Custom font subset (fontello/icomoon)
Gera TTF customizado com só os glifos usados. Esforço ~2h. Economia ~4 MB. Risco médio (manutenção do pipeline).

## Aplicação no projeto
- **Decisão (2026-05-16)**: Opção A. Adicionado ao backlog Fase 2 como task low-priority em sprint T2.1 ou polish entre sprints. Fase 2 já tem escopo intenso; spike fix imediato fica para janela de oportunidade.

## Como evitar (futuro)
- NUNCA importar de `@expo/vector-icons` raiz; preferir `lucide-react-native` (já no projeto)
- Se uma família específica de vector-icons for absolutamente necessária, usar import específico do path (`@expo/vector-icons/Ionicons`)
- Considerar adicionar ESLint rule `no-restricted-imports` proibindo `@expo/vector-icons` (sem `/Familia` no path)
