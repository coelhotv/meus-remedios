# [AP-157] Normalizar dataset grande (NFD) em cada keystroke vs pré-normalizar no boot

## Sintoma
Hook de busca cliente-side sobre dataset grande (~6800 registros ANVISA no Dosiq) normalizava o texto (`String.prototype.normalize('NFD')`) **dentro do loop de busca** — uma vez por registro, por keystroke. Em devices Android low-end, isso causou jank perceptível em listas autocomplete.

## Correção
Pré-normalizar **uma vez** quando o dataset carrega, via `useMemo([database])`:

```javascript
// Errado — normalize por keystroke
function search(query) {
  const q = normalizeText(query)
  return database.filter(m => normalizeText(m.name).includes(q))
}

// Correto — pré-normaliza no boot
const normalizedDatabase = useMemo(
  () => database.map(m => ({
    med: m,
    nName: normalizeText(m.name),
    nIngredient: normalizeText(m.active_ingredient),
  })),
  [database]
)
function search(query) {
  const q = normalizeText(query)
  return normalizedDatabase.filter(({ nName }) => nName.includes(q))
}
```

## Impacto
- Sem pré-normalização: 6800 × `normalize('NFD')` por keystroke = jank.
- Com pré-normalização: 6800 × 1x no boot, depois filter puro.

## Sinal de detecção
Gemini-bot tipicamente sinaliza como "performance — re-cálculo em renders". É verdade. Aplicar useMemo no dataset, não no resultado.
