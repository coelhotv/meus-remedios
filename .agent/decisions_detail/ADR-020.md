# ADR-020: Separate Stock Purchases from Inventory Tracking

**Status:** Accepted | **Date:** 2026-04-02 | **Source:** PR #443
**Tags:** stock, inventory, refactor

## Context
Stock purchases and inventory tracking logic were intertwined, causing data inconsistencies and bugs.

## Decision
Separate purchases and inventory into **distinct domains**:
- **Purchases:** Track when/what was bought (supplier, cost, date)
- **Inventory:** Track current stock levels and consumption (quantity, dosage, usage)

Each has independent data models and services.

## Implementation
```
src/features/stock/
├── purchases/      # Purchase orders
├── inventory/      # Stock tracking
└── stockService.js # Unified interface
```

## Consequences
✅ Clear separation of concerns
✅ Reduced data conflicts
✅ Easier to audit purchases
⚠️ More data models to maintain
⚠️ Complex inventory calculations
