# ADR-007: Wave-Based Redesign Incremental Delivery

**Status:** Accepted
**Date:** 2026-03-24
**Source:** PR #416, PR #417-425
**Tags:** redesign, architecture, waves

## Context

Complete visual redesign needed (Santuário Terapêutico) affecting typography, colors, components, and layouts. Rolling this out all-at-once would be risky and difficult to validate.

## Decision

Break redesign into **15 sequential waves**, each with discrete scope:

- **Wave 0:** Design Tokens (colors, shadows, spacing)
- **Wave 1:** Typography + Icon System
- **Wave 2:** Surface & Layout System
- **Wave 3:** Component Primitives
- **Waves 4-12:** Feature-specific redesigns (Navigation, Dashboard, Medicines, etc.)
- **Waves 13-15:** Polish, accessibility, final validation

Each wave:
- Is independently testable
- Can be feature-flagged
- Has dedicated PR with clear scope
- Includes design specs and implementation details

## Consequences

✅ Reduces risk by validating each wave independently
✅ Allows parallel work on different features
⚠️ Requires 15+ PRs and careful dependency management
⚠️ Team must understand wave dependencies

## Testing

Each wave has dedicated PR with comprehensive testing:
- Visual regression tests
- Accessibility checks (WCAG 2.1 AA)
- Component story validation
