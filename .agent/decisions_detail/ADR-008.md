# ADR-008: Design System Hierarchy: Tokens → Typography → Components → Layouts

**Status:** Accepted | **Date:** 2026-03-24 | **Source:** PR #417
**Tags:** design-system, tokens, architecture

## Context
Need structured design system foundation to ensure consistency and enable efficient implementation across redesign waves.

## Decision
Implement four-level design system hierarchy:
1. **Wave 0:** Design Tokens (colors, shadows, spacing, border radius)
2. **Wave 1:** Typography + Icon System
3. **Wave 2:** Surface & Layout System
4. **Wave 3:** Component Primitives

Each level builds on previous, ensuring consistency.

## Implementation
- `tokens.redesign.css` — CSS variables for all tokens
- `typography.redesign.css` — font stacks, sizes, weights
- `layout.redesign.css` — grid, spacing, responsive rules
- `components.redesign.css` — button, card, form styles

## Consequences
✅ Consistent design across app
✅ Easy to maintain and update
✅ Clear dependency hierarchy
⚠️ All waves must follow hierarchy
⚠️ Tight coupling between waves
