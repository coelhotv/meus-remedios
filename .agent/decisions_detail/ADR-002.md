# ADR-002: Environment-Aware Logger to Reduce Console Noise

**Status:** Accepted | **Date:** 2026-03-21 | **Source:** PR #413
**Tags:** logging, debugging, refactor

## Context
Dashboard console was generating excessive debug logs, making it difficult to debug issues.

## Decision
Implement environment-aware logger that suppresses debug output in production while maintaining full logs in development.

## Implementation
- Check `process.env.NODE_ENV !== 'production'` before console output
- Categorize logs: DEBUG, INFO, WARN, ERROR
- Only emit DEBUG level in development

## Consequences
✅ Cleaner production console
✅ Full debug visibility in development
⚠️ Must configure logging in all modules

## Testing
Verify console output differs between development and production builds.
