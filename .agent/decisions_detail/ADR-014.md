# ADR-014: Feature-Based Architecture Over Layer-Based

**Status:** Accepted | **Date:** 2026-03-23 | **Source:** PR #440
**Tags:** architecture, code-organization

## Context
Original layer-based architecture (src/components, src/hooks, src/utils) made it difficult to locate related code for a feature.

## Decision
Reorganize codebase to feature-based architecture:
```
src/features/
├── medicines/
│   ├── components/
│   ├── hooks/
│   ├── services/
│   └── schemas/
├── adherence/
│   ├── components/
│   ├── hooks/
│   └── services/
└── ...
```

## Consequences
✅ Related code for feature in one place
✅ Easier to understand feature scope
✅ Simpler to delete/deprecate features
⚠️ Shared code must go to `src/shared/`
⚠️ Requires discipline to not create circular imports

## Testing
Run ESLint no-restricted-imports to enforce boundaries.
