# Release Notes — v4.0.0

**Data:** 2026-04-02
**Tipo:** Major
**Status:** Pré-merge aprovado

## Resumo

A `v4.0.0` introduz o maior refactor já feito no domínio de estoque do Meus Remédios. O histórico de compras foi separado do saldo corrente, o consumo FIFO passou a ser transacional e rastreável por lote, e o fluxo redesign (`?redesign=1`) virou a superfície oficial dessa evolução.

Além disso, a release incorpora `regulatory_category` da ANVISA para suportar novas regras de UX na compra de medicamentos e alinha o bot Telegram ao mesmo modelo de estoque da aplicação web.

## Principais mudanças

### Estoque e compras

- criação da tabela `purchases` como fonte canônica do histórico de compras
- evolução de `stock` para saldo corrente por lote
- criação de `stock_adjustments` para auditoria de correções e restaurações
- criação de `stock_consumptions` para rastrear exatamente quais lotes cada `medicine_log` consumiu
- backfill idempotente do modelo legado

### RPCs transacionais

- `create_purchase_with_stock`
- `consume_stock_fifo`
- `restore_stock_for_log`
- `apply_manual_stock_adjustment`

### Redesign

- histórico e última compra passam a ser lidos de `purchases`
- `StockForm` recebe `pharmacy`, `laboratory` e `notes`
- regra de laboratório por compra passa a depender de `regulatory_category`

### ANVISA

- `regulatory_category` incorporado em `medicines`
- ETL e formulários passam a preservar essa informação

### Telegram

- `/adicionar_estoque` cria compra + lote via RPC
- registro de dose usa FIFO no banco, sem mutação manual de `stock`

- a superfície funcional alvo desta release é o redesign `?redesign=1`
- a UI legacy pode continuar operacional por compatibilidade técnica, mas não é o critério de aceite principal desta release
